考點：SOC 報告判斷

Exam: SOC Report Identification
01 — 考前重點濃縮

本封考SOC 報告判斷:SOC 1/2/3、Type I vs II(高頻)。評估供應商該看哪份。

SOC 1=財報內控;SOC 2=資安(保密);SOC 3=公開摘要。
Type I=某時間點設計;Type II=一段期間運作有效(證據力強)。
評估供應商資安→SOC 2 Type II。
02 — 核心速記
報告內容
SOC 1財報內控(給財務稽核員)
SOC 2資安/信任服務(保密)
SOC 3SOC 2 公開摘要
Type I/II時間點設計/期間運作有效
03 — 關鍵名詞（中英）
SOC 1/2/3（服務組織控制報告）— 財報/資安/公開。
Type I（型一）— 某時間點設計適當性。
Type II（型二）— 一段期間運作有效性。
Trust Services Criteria（信任服務準則）— SOC 2 五準則。
CUEC（使用者實體控制）— 客戶自身須做的控制。
04 — 常見陷阱

SOC 1≠資安(是財報);評估資安看 SOC 2。

Type II>Type I(期間 vs 時間點);供應商選 Type II。

別忽略 CUEC(你自己也要做的控制)。

05 — 🎯 實戰考題
QUESTION 1

A company is evaluating a SaaS vendor's SECURITY controls and wants evidence they operated effectively over the past year. Which report is MOST appropriate?

（某公司要評估 SaaS 供應商的『資安』控制,並想要過去一年運作有效的證據。哪份報告最適合?）

A. SOC 1 Type I
B. SOC 2 Type II
C. SOC 3
D. SOC 1 Type II
✅ 解答與解析

正解：B。評估資安→SOC 2;要一段期間運作有效的證據→Type II。故 SOC 2 Type II。

為何其他選項錯：
  • A：SOC 1 是財報內控,且 Type I 只是時間點快照。
  • C：SOC 3 只是公開摘要,細節不足以評估。
  • D：SOC 1 是財報,非資安。
QUESTION 2

What is the key difference between a SOC 2 Type I and Type II report?

（SOC 2 Type I 與 Type II 報告的關鍵差異是什麼?）

A. Type I covers finances, Type II covers security 型一財務、型二資安
B. Type I assesses design at a point in time; Type II assesses operating effectiveness over a period 型一某時間點設計、型二一段期間運作有效
C. Type I is public, Type II is private 型一公開、型二保密
D. There is no difference 沒有差別
✅ 解答與解析

正解：B。Type I＝某時間點控制『設計是否適當』;Type II＝一段期間控制『是否持續有效運作』(證據力強)。

為何其他選項錯：
  • A：兩者都是 SOC 2(資安),差別不在財務 vs 資安。
  • C：兩者都屬保密報告。
  • D：兩者差別明確(設計 vs 運作)。
QUESTION 3

In a SOC 2 report, controls that the customer themselves must implement for the overall system to be secure are called:

（在 SOC 2 報告中,『客戶自己必須落實、整個系統才安全』的控制稱為:）

A. Complementary User Entity Controls (CUECs) 使用者實體控制
B. Trust Services Criteria 信任服務準則
C. Exceptions 例外事項
D. Management assertions 管理層聲明
✅ 解答與解析

正解：A。CUEC 是『客戶端須自行落實的補充控制』;有供應商 SOC 2 不代表你全免責,仍須做好 CUEC。

為何其他選項錯：
  • B：信任服務準則是 SOC 2 評估的五大面向。
  • C：例外事項是稽核發現的控制失效。
  • D：管理層聲明是服務商對控制的聲明。
06 — (ISC)² 選最佳答案思路

評估資安→SOC 2 Type II;財報→SOC 1;公開摘要→SOC 3。

Type II(期間)>Type I(時間點);讀報告別漏 CUEC 與例外事項。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：SOC 1 財報/2 資安/3 公開;Type I 時間點設計、Type II 期間運作有效;評估供應商資安看 SOC 2 Type II。

Q1 . 評估供應商資安且要一年運作證據,看哪份？
→ SOC 2 Type II。

Q2 . Type I 與 Type II 差別？
→ 型一某時間點設計;型二一段期間運作有效性。

Q3 . 客戶自己須落實的補充控制叫什麼？
→ CUEC(使用者實體控制)。

📑 官方考綱對照重點（Official Objectives · §6.5 SOC 報告判斷（考點））
▍SOC 選擇

要給審計師看財報相關內控→SOC 1；要評估服務商資安/可用/隱私（給利害關係人，含細節機密）→SOC 2；要公開行銷用摘要→SOC 3。

只看某時點設計→Type I；要一段期間運作有效性（保證較高）→Type II。

對應官方 §6.5（SOC 報告判斷，考點）。

§6.5.3 Third-party
• Third-party audits are often needed to demonstrate compliance with some government regulation or industry standard
• Advantages of having a third-party audit an organization:
• they likely have breadth of experience auditing many types of systems, across many types of organizations
• they are not affected by internal dynamics or org politics
• Disadvantage of using a third-party auditor:
• cost: third-party auditors are going to be much more costly than internal teams; this means that the organization is not likely to conduct audits as frequently
• internal resources are still required to assist or accompany auditors, to answer questions and guide
• Some common third-party audit frameworks include:
• ISO 27001 (certification audits)
• PCI DSS (for payment card data)
• HIPAA (for healthcare)
