考點：弱點 vs 滲透

Exam: Vuln vs Pentest
01 — 考前重點濃縮

D6 後期。本封考弱點評估 vs 滲透測試的差異與時機(最常考),以及黑/白/灰箱判斷。

弱點評估=找出&列出弱點(廣、不利用);滲透測試=實際利用證明可被攻破(深)。
黑箱(無內部資訊)/白箱(全知)/灰箱(部分)。
滲透測試必須先授權(ROE/書面)。
02 — 核心速記
面向弱點評估滲透測試
目的找&列弱點實際利用證明
深度廣、表面深、針對
前提—須書面授權/ROE
03 — 關鍵名詞（中英）
Vulnerability Assessment（弱點評估）— 找出並分級弱點。
Penetration Test（滲透測試）— 實際利用弱點。
Black/White/Gray box（黑/白/灰箱）— 資訊掌握多寡。
ROE（交戰規則）— 範圍/授權。
Red Team（紅隊）— 目標導向對抗演練。
04 — 常見陷阱

評估(找) vs 滲透(打):預算有限先評估。

滲透須先授權(未授權=非法)。

紅隊測整體偵測回應,範圍比滲透更廣。

05 — 🎯 實戰考題
QUESTION 1

Management wants to know which known weaknesses exist across all servers, but does NOT want any exploitation. Which activity is MOST appropriate?

（管理層想知道所有伺服器上存在哪些已知弱點,但『不希望進行任何利用』。哪個活動最適合?）

A. Penetration test 滲透測試
B. Vulnerability assessment 弱點評估
C. Red team engagement 紅隊演練
D. Social engineering test 社交工程測試
✅ 解答與解析

正解：B。要『找出並列出弱點而不利用』＝弱點評估(廣度掃描、不 exploit)。

為何其他選項錯：
  • A：滲透測試會實際利用弱點,違反『不利用』要求。
  • C：紅隊會模擬攻擊並利用,範圍更廣。
  • D：社交工程是利用人,非列舉系統弱點。
QUESTION 2

A penetration tester is given NO prior information about the target environment, simulating an external attacker. This is known as:

（滲透測試人員未獲任何目標環境資訊,模擬外部攻擊者。這稱為:）

A. White box 白箱
B. Gray box 灰箱
C. Black box 黑箱
D. Crystal box 透明箱
✅ 解答與解析

正解：C。無任何內部資訊、模擬外部攻擊者＝黑箱(black box)。

為何其他選項錯：
  • A：白箱是完全掌握架構/原始碼。
  • B：灰箱是部分資訊。
  • D：transparent/crystal box 即白箱別稱(全知),非無資訊。
QUESTION 3

What MUST be obtained before conducting a penetration test to ensure it is legal?

（進行滲透測試前,為確保合法『必須』取得什麼?）

A. A vulnerability scan report 弱點掃描報告
B. Written authorization and rules of engagement 書面授權與交戰規則
C. A SOC 2 report SOC 2 報告
D. An incident response plan 事件回應計畫
✅ 解答與解析

正解：B。滲透測試必須有書面授權(get-out-of-jail letter)與 ROE(界定範圍/時間/手段),否則等同非法入侵。

為何其他選項錯：
  • A：掃描報告非合法前提。
  • C：SOC 2 與滲透授權無關。
  • D：IR 計畫不是滲透測試的法律授權。
06 — (ISC)² 選最佳答案思路

找&列=弱點評估;實際利用=滲透;無資訊=黑箱。滲透先授權。

預算/時間有限先做弱點評估;要證明真實風險才滲透;測整體韌性用紅隊。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：弱點評估找&列(不利用)、滲透實際利用(須授權);黑箱無資訊/白箱全知/灰箱部分。

Q1 . 只找弱點不利用,用哪種活動？
→ 弱點評估。

Q2 . 模擬無內部資訊的外部攻擊者是哪種箱型？
→ 黑箱(black box)。

Q3 . 滲透測試合法的必要前提？
→ 書面授權與 ROE。
