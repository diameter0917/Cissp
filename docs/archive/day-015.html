實體與場所安全

Physical & Site Security
01 — 別忘了實體世界

再強的加密也擋不住「有人把伺服器搬走」或「機房淹水斷電」。實體與場所安全（Physical Security）是 Domain 3 的收尾，也是常被技術人輕忽的考點。今天看場所設計（CPTED）、電力、環境/空調、滅火、機房分級。

🗺️ 本週回顧：①設計原則 → ②安全模型 → ③密碼學 → ④系統/雲端弱點 → ⑤今天：實體與場所安全。下週進入 Domain 4（通訊與網路安全）。
02 — CPTED 與分層實體防護

CPTED（Crime Prevention Through Environmental Design，以環境設計預防犯罪）透過三概念降低犯罪：自然監視（視線開闊）、自然存取控制（動線引導）、領域感（明確界線）。實體防護同樣採縱深：圍籬 → 門禁 → 警衛/CCTV → 上鎖機櫃層層套疊。

03 — 電力與環境
電力：UPS 撐短時間、發電機撐長時間；認識 brownout（持續低電壓）/ blackout（停電）/ surge（突波）
環境/空調（HVAC）：控溫控濕；機房常用正壓防塵、冷熱通道（hot/cold aisle）配置散熱
偵測：煙霧/溫度/漏水感測，及早告警
04 — 滅火系統
類型適用/重點
Wet pipe 濕管管內常存水，最常見；但機房漏水風險
Dry pipe / Pre-action 乾管/預動平時無水，降低誤觸漏水，適合機房
Gas（如 FM-200） 氣體不傷設備、適合機房；須注意人員氧氣
Class 分級電器火災屬 Class C（美），須用不導電滅火劑

💡 機房滅火優先考量「不傷設備＋人員安全」：常用氣體系統或預動式；電器火災不可用水或一般泡沫。

05 — 關鍵名詞速查
CPTED（環境設計預防犯罪）— 自然監視/存取控制/領域感降低犯罪。
UPS（不斷電系統）— 停電時短時供電，配合發電機。
Brownout（持續低電壓）— 電壓不足，可能損壞設備。
Hot/Cold Aisle（冷熱通道）— 機房散熱配置，提升冷卻效率。
Mantrap / Access Control Vestibule（雙門互鎖）— 一次一人通過，防尾隨（tailgating）。
Fire Suppression（滅火系統）— 濕管/乾管/氣體；機房重視不傷設備與人安。
06 — 易混淆釐清

Fail-safe vs Fail-secure（門禁）：火災等人命情境，門應 fail-safe（斷電即開讓人逃生）；高安全資產區可能要 fail-secure（斷電即鎖）。看『保命 vs 保資產』取捨。

Blackout vs Brownout：blackout＝完全停電；brownout＝持續電壓不足。兩者都靠 UPS/發電機緩解。

07 — 條文與標準

機房可參考 Uptime Institute Tier I–IV 分級（冗餘度遞增）；NIST SP 800-53 的 PE（Physical & Environmental）控制族涵蓋實體與環境要求；消防分級參考 NFPA。

08 — 真實事件：機房火災/斷電
🔎 雲端也怕火與電

2021 年某大型歐洲資料中心發生嚴重火災，導致大量客戶網站與資料中斷，部分未異地備份的客戶永久失去資料——再次證明可用性（A）與實體/環境控制是真實風險，不是紙上談兵。電力與冷卻一旦失效，整個機房可在數分鐘內停擺。

教訓：實體安全＝CIA 的可用性地基；異地備份、冗餘電力/冷卻、滅火設計缺一不可。呼應 Domain 1 的 BCP/DR。

09 — CISSP 考點提醒

選最佳答案：「防尾隨進入機房」→ mantrap/雙門互鎖；「機房滅火不傷設備」→ 氣體系統；「停電維持運作」→ UPS＋發電機。

常見陷阱：涉及人命時門禁應 fail-safe（開門逃生）而非鎖死；電器火災（Class C）不可用水。實體控制常是『最佳答案』裡被忽略的那個。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：實體安全守 CIA 的可用性：CPTED 設計、分層門禁、UPS/發電機、HVAC 與冷熱通道、機房用氣體/預動滅火，並異地備份。

Q1 . 防尾隨（tailgating）最有效的實體控制？
→ Mantrap／雙門互鎖（access control vestibule）。

Q2 . 機房滅火為何偏好氣體系統？
→ 不傷電子設備、滅火快；但須注意人員氧氣。

Q3 . 火災時資料中心大門該 fail-safe 還 fail-secure？
→ Fail-safe（斷電開門）以保人命逃生。
