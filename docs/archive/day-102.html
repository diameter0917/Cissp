考點：存取控制模型

Exam: Access Control Models
01 — 考前重點濃縮

本封考存取控制模型 DAC/MAC/RBAC/ABAC 的情境判斷——高頻考點。判斷『誰/依什麼決定權限』。

DAC:擁有者自行授權(彈性、易擴散);MAC:系統依標籤強制(軍方、不可自改)。
RBAC:依角色(企業常用);ABAC:依屬性/情境(零信任、最細緻)。
02 — 模型速記
模型誰/依什麼決定
DAC擁有者自行授權
MAC系統依標籤強制(不可自改)
RBAC依角色
ABAC依屬性/情境(時間/位置/裝置)
03 — 關鍵名詞（中英）
DAC（自主存取控制）— 擁有者授權。
MAC（強制存取控制）— 系統依標籤強制(軍方)。
RBAC（角色存取控制）— 依角色授權。
ABAC（屬性存取控制）— 依屬性/情境動態決策。
Rule-based（規則型）— 依全域規則(如 ACL)。
04 — 常見陷阱

DAC(擁有者) vs MAC(系統強制):MAC 使用者不能自改。

RBAC(角色) vs ABAC(屬性/情境):零信任偏 ABAC。

MAC 最嚴、DAC 最彈性、ABAC 最細緻。

05 — 🎯 實戰考題
QUESTION 1

In a system where the data owner can grant other users access to files they own at their own discretion, which access control model is in use?

（在一個『資料擁有者可自行決定授予他人存取自己檔案』的系統中,使用哪種存取控制模型?）

A. MAC 強制存取控制
B. DAC 自主存取控制
C. RBAC 角色存取控制
D. ABAC 屬性存取控制
✅ 解答與解析

正解：B。擁有者自行裁量授權＝DAC(自主);彈性但易導致權限擴散。

為何其他選項錯：
  • A：MAC 由系統依標籤強制,擁有者不能自行授權。
  • C：RBAC 依角色,非擁有者裁量。
  • D：ABAC 依屬性/情境策略。
QUESTION 2

A military system labels every object and subject with a classification, and the OS strictly enforces access based on these labels regardless of user wishes. This is:

（某軍事系統為每個物件與主體標上密級,作業系統依標籤嚴格強制存取,不論使用者意願。這是:）

A. DAC
B. MAC 強制存取控制
C. RBAC
D. Rule-based
✅ 解答與解析

正解：B。依標籤由系統強制、使用者不可自改＝MAC,典型用於軍事/高機密環境。

為何其他選項錯：
  • A：DAC 由擁有者裁量,非系統強制。
  • C：RBAC 依角色,非密級標籤。
  • D：規則型依全域規則,非密級標籤強制。
QUESTION 3

An organization grants access dynamically based on attributes such as department, time of day, device compliance, and location. Which model is this?

（某組織依部門、時段、裝置合規、位置等屬性動態授予存取。這是哪種模型?）

A. DAC
B. MAC
C. RBAC
D. ABAC 屬性存取控制
✅ 解答與解析

正解：D。依多種屬性/情境動態決策＝ABAC,最細緻,常用於零信任。

為何其他選項錯：
  • A／B：DAC/MAC 不依情境屬性動態決策。
  • C：RBAC 依靜態角色,不涵蓋時段/裝置/位置等情境。
06 — (ISC)² 選最佳答案思路

口訣:擁有者裁量=DAC;系統依標籤強制=MAC(軍方);依角色=RBAC;依屬性情境=ABAC(零信任)。

看到『不論使用者意願、系統強制』選 MAC;『部門/時段/裝置』選 ABAC。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：DAC 擁有者裁量、MAC 系統依標籤強制(軍方)、RBAC 依角色、ABAC 依屬性情境(零信任)。

Q1 . 擁有者自行決定授權是哪種模型？
→ DAC(自主存取控制)。

Q2 . 軍方依標籤、系統強制、使用者不可改是哪種？
→ MAC(強制存取控制)。

Q3 . 依部門/時段/裝置/位置動態授權是哪種？
→ ABAC(屬性存取控制)。
