資料角色與擁有權

Data Roles & Ownership
01 — 資料的「人」：誰對資料負責

昨天我們替資料貼了分類標籤；今天要釐清圍繞資料的各種角色——誰決定怎麼用、誰實際保管、誰的資料、誰代為處理。這在 CISSP 極常考，且與 GDPR 的法律角色直接對應。把角色分清楚，責任歸屬與考題就都迎刃而解。

🗺️ 今天重點：六個資料角色一句話 → 與 GDPR 對應 → controller vs processor（最常考）→ 責任歸屬。
02 — 資料角色一覽
角色 負責什麼 GDPR 對應
Data Owner
資料擁有者		決定分類、定義保護需求與用途（高階）≈ Controller 內部代表
Data Controller
資料控管者		決定「為何、如何」處理個資（法律主體）Controller
Data Custodian
保管者		依 Owner 要求執行備份、加密、存取設定（IT）—（技術執行）
Data Processor
處理者		代 Controller 處理個資（如雲端/外包商）Processor
Data Subject
資料當事人		資料所描述的本人（享有權利）Data Subject
Data Steward
資料管家		負責資料品質與意涵（business 面）—
03 — 最常考：Controller vs Processor
Controller
決定「為何/如何」
（出題目的的人）		 委託→ Processor
照指示處理
（執行的人）

一句話判斷：決定目的與方式的是 Controller；只是照單執行的是 Processor。例如一家公司用某雲端 CRM 寄行銷信——公司是 Controller、CRM 廠商是 Processor。GDPR 下兩者都有責任，但 Controller 負主要義務（合法性基礎、告知當事人），Processor 須依約處理並協助。

04 — 關鍵名詞速查
Data Owner（資料擁有者）— 通常為業務主管，決定分類與使用，當責不可下授。
Data Controller（控管者）— 決定個資處理目的與方式的法律主體。
Data Processor（處理者）— 代控管者處理個資者，須依合約與指示。
Data Custodian（保管者）— IT/DBA，依擁有者要求執行技術保護。
Data Subject（當事人）— 個資所描述之本人，享查詢/更正/刪除等權利。
DPO（資料保護長）— Data Protection Officer，GDPR 特定情形下須設置，監督法遵。
05 — 易混淆釐清

Owner vs Custodian：Owner 決定（分類、誰可存取）；Custodian 執行（備份、加密、設定權限）。Owner 是業務、Custodian 是 IT。

Controller vs Owner：Owner 是組織內部的資料負責角色；Controller 是法律上對個資負責的主體（常是整個組織）。考個資法/GDPR 時用 Controller/Processor 語彙，考內控時用 Owner/Custodian。

06 — 條文與標準

GDPR 第 4 條定義 controller / processor / data subject；第 28 條要求 Controller 與 Processor 間須簽資料處理協議（DPA）。台灣個資法區分「公務／非公務機關」為蒐集處理利用之主體，概念與 Controller 相近。ISO/IEC 27701（隱私資訊管理）則把這些角色責任制度化。

07 — 真實事件：Controller vs Processor 的責任
🔎 外包商出包，誰要負責？

當企業（Controller）把個資交給外包商（Processor）處理、外包商因疏失導致外洩時，GDPR 的設計是兩者都可能被追責：Controller 因「未善盡選任與監督」、Processor 因「未依約落實安全措施」。這也是為什麼第 28 條強制要簽 DPA、載明安全要求與通報義務。

對照 Domain 1 的供應鏈風險：把資料交出去，責任不會跟著一起交出去。盡職調查＋合約＋稽核權是 Controller 自保的關鍵。

08 — CISSP 考點提醒

高頻考點：情境給「某公司委託 SaaS 廠商處理客戶名單」，問廠商是什麼角色 → Processor；公司是 Controller。記「決定目的＝Controller、照做＝Processor」。

常見陷阱：別把 Owner（內部、決定分類）和 Custodian（IT、執行）搞反；也別把 Controller 的法律責任推給 Processor——Controller 永遠負主要義務。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：Owner 決定、Custodian 執行；GDPR 下 Controller 定目的、Processor 照做，兩者皆有責、Controller 負主要義務。

Q1 . 雲端 CRM 廠商代客戶寄行銷信，廠商是什麼角色？
→ Processor（處理者）；委託它的公司是 Controller。

Q2 . 決定資料分類的是 Owner 還是 Custodian？
→ Data Owner（業務）；Custodian 只負責技術執行。

Q3 . GDPR 要求 Controller 與 Processor 間簽什麼？
→ DPA 資料處理協議（第 28 條），載明安全與通報義務。
