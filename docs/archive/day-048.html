資料生命週期與落地主權

Lifecycle, Sovereignty & Residency
01 — 深入生命週期、最小化與資料主權

中期深入資料的『流動』面：蒐集最小化、資料的位置/主權/落地、以及雲端帶來的跨境難題。雲端讓資料『不知道在哪』，但法律責任仍依資料所在地與供應商母國而定。

🗺️ 今天重點：資料最小化 → residency/sovereignty/localization → 雲端共同責任與跨境 → 保存 vs 最小化衝突。
02 — 資料最小化與限期
Collection Limitation 蒐集最小化：只蒐集特定目的所需（隱私核心原則）
Purpose Limitation 目的拘束：不得逾原蒐集目的另作他用
Storage Limitation 限期保存：目的達成即刪除（呼應 retention）
💡 最少資料＝最小攻擊面與法遵風險——『沒蒐集的資料不會外洩』
03 — Residency / Sovereignty / Localization
概念定義關注
Residency 存放地資料實際儲存的地理位置選 region
Sovereignty 主權資料受所在地（與供應商母國）法律管轄誰能合法調取
Localization 在地化法規強制個資留境內合規義務

關鍵洞見：存放地 ≠ 管轄權。資料存在歐盟的美商雲，仍可能因美國 CLOUD Act 被美國政府要求調取——這正是 Schrems II 爭議的核心。選雲端 region 是法律決策。

04 — 雲端共同責任與跨境

共同責任模型（Shared Responsibility）：雲商負責『雲的安全（of the cloud）』、客戶負責『雲中的安全（in the cloud）』——資料分類、存取控制、加密金鑰、合規多半仍是客戶責任。跨境傳輸（D1 已述）需 adequacy/SCC/BCR。把個資放雲前，先確認 region、子處理者清單與資料調取條款。

05 — 關鍵名詞速查
Data Minimization（資料最小化）— 只蒐集目的所需的最少資料。
Purpose Limitation（目的拘束）— 不得逾原目的使用個資。
Data Residency（資料存放地）— 資料實際儲存的位置。
Data Sovereignty（資料主權）— 資料受所在地/供應商母國法律管轄。
Data Localization（在地化）— 法規強制資料留境內。
Shared Responsibility（共同責任）— 雲商與客戶各負其責的模型。
Subprocessor（子處理者）— Processor 再委託的第三方。
06 — 易混淆釐清

Residency vs Sovereignty：Residency＝存在哪（地理）；Sovereignty＝受誰管（法律）。資料可存在 A 國卻受 B 國法律影響（供應商母國）。

雲端責任：『用了雲＝雲商負責安全』是錯誤。資料、存取、金鑰、合規多半仍是客戶責任（共同責任模型）。

Retention vs Minimization：法規要求最低保存年限（財務/醫療）可能與個資目的達成即刪衝突——須逐類資料分別訂定，不能一刀切。

07 — 條文與標準

OECD 隱私原則（蒐集限制、目的明確、使用限制）、GDPR Art.5（最小化/限期）、台灣個資法（特定目的、目的消失刪除）。雲端跨境另涉 CLOUD Act、Schrems II 後的 SCC＋補充措施。

08 — 真實事件：雲端資料主權爭議
🔎 資料在歐洲，卻被外國法律觸及

歐洲組織把個資放在美商雲的歐盟 region，原以為『資料在歐盟就合規』，卻因供應商受美國 CLOUD Act 管轄、理論上美國政府可要求調取，引發主權疑慮與合規挑戰；Schrems II 後更須額外的 SCC 與技術補充措施（如客戶持金鑰加密）。促成歐洲『主權雲（sovereign cloud）』需求。

教訓：資料主權看『受誰管轄』而非『存在哪』；雲端選型是法律＋技術＋商業的綜合決策，加密金鑰由客戶掌握可降低被第三方調取的風險。

📌 延伸：刪除在雲端與備份的難題

『被遺忘權』要求刪除，但雲端資料常散落於多副本、備份、子處理者。真正落實刪除需要：可定位所有副本、納入備份輪替的刪除、要求子處理者連動刪除。crypto-shredding（銷毀金鑰）常用於『邏輯刪除』大量加密資料。

09 — CISSP 考點提醒

選最佳答案：『資料受哪國法律管轄』→ 主權；『法規要求留境內』→ 在地化；『用雲後誰負責資料分類/加密』→ 客戶（共同責任）。

常見陷阱：存放地≠管轄權；雲商不負責你的資料保護全部；retention 法定年限可能與最小化衝突，需分類別訂定。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：最小化/目的拘束/限期保存降風險；residency(存哪)≠sovereignty(受誰管)；雲端共同責任下資料/金鑰/合規多屬客戶；跨境需合法機制。

Q1 . 資料存在歐盟的美商雲，主權上仍受誰影響？
→ 供應商母國法律（如美國 CLOUD Act）——主權看管轄非存放地。

Q2 . 雲端共同責任模型下，資料分類與加密金鑰通常誰負責？
→ 客戶（in the cloud 的安全）。

Q3 . 法定最低保存年限與個資最小化刪除衝突時怎麼辦？
→ 依資料類別分別訂定保存政策，不可一刀切。
