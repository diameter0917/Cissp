資料角色與責任深入

Data Roles Deep Dive
01 — 深入資料角色的法律與營運責任

初期我們認識了 owner/custodian/controller/processor/subject；中期深入各角色的法律與營運責任、用 RACI 釐清權責、處理 joint controllers（共同控管者） 與 DPO，並把這些角色與隱私法的義務銜接。

🗺️ 今天重點：角色法律/營運責任 → RACI → joint controller/DPO → 與 GDPR/個資法銜接 → 責任歸屬。
02 — 角色責任（深入）
角色營運責任法律責任重點
Data Owner分類、定義保護需求、核准存取對資料保護當責（內部）
Data Controller決定處理目的與方式GDPR 主要義務：合法性、告知、回應當事人
Data Processor依指示處理須依 DPA、落實安全、協助 Controller、通報
Data Custodian執行技術保護依 Owner 要求落實，無決策權
Data Subject—享查詢/更正/刪除/可攜/反對等權利
03 — RACI 釐清權責

RACI（Responsible 執行／Accountable 當責／Consulted 諮詢／Informed 告知）是釐清『誰做、誰扛、誰問、誰知』的工具。資安治理常見錯誤是『多個 A』或『沒有 A』——當責必須唯一且明確（呼應 D1：accountability 不可下授、不可分散）。

04 — Joint Controller 與 DPO
Joint Controllers 共同控管者：兩方共同決定處理目的與方式（如合作行銷），須以協議劃分各自責任
Controller–Processor：一方決定、一方執行，須簽 DPA（GDPR Art.28）
DPO 資料保護長：GDPR 特定情形（大規模特種個資/系統性監控/公務機關）須設置，獨立監督法遵、為當事人與主管機關聯絡窗口
05 — 關鍵名詞速查
Data Controller（控管者）— 決定個資處理目的與方式（法律主體）。
Data Processor（處理者）— 代控管者處理，依 DPA 與指示。
Joint Controllers（共同控管者）— 共同決定目的方式，需協議分責。
DPO（資料保護長）— 獨立監督個資法遵的職位。
RACI（權責矩陣）— Responsible/Accountable/Consulted/Informed。
DPA（資料處理協議）— Controller-Processor 間的合約義務。
Liability（法律責任）— 未善盡義務需承擔的賠償責任。
06 — 易混淆釐清

Responsible vs Accountable（RACI）：Responsible 是實際做的人（可多人）；Accountable 是最終扛責的人（必須唯一）。對應 D1：責任可分、當責不可。

Joint Controllers vs Controller-Processor：前者都在決定目的（平行）；後者一方決定、一方執行（上下）。判準是『誰決定為何處理』。

Owner vs Controller：Owner 是組織內部角色；Controller 是法律主體（常是組織本身）。內控語境用 Owner，法遵語境用 Controller。

07 — 條文與標準

GDPR Art.4/24/26/28/37-39（角色定義、Controller 責任、Joint Controller、Processor、DPO）；台灣個資法以公務/非公務機關為責任主體、課保護義務；ISO/IEC 27701 把角色責任制度化（PII Controller/Processor）。

08 — 真實事件：角色責任爭議
🔎 外包出包，互踢皮球

個資外洩後，企業（Controller）與外包商（Processor）常互推責任：企業說『是廠商系統的漏洞』，廠商說『我們只是照指示做』。GDPR 的設計讓兩者都可能被追責——Controller 因選任監督不周、Processor 因未落實約定安全。關鍵證據是DPA 有沒有簽、責任有沒有寫清楚。

教訓：角色與責任要事前以合約（DPA）＋RACI 釐清，而非事後爭論。Controller 無法靠『外包』卸責。

📌 延伸：員工是什麼角色？

常見混淆：一般員工存取個資時是什麼角色？員工是代表組織（Controller）處理資料的授權使用者（user），不是獨立的 Controller/Processor。責任歸於組織。但員工逾越授權濫用個資（如偷查名人病歷），則可能自負個人法律責任。

09 — CISSP 考點提醒

選最佳答案：『兩公司共同決定處理目的』→ Joint Controllers；『RACI 中誰必須唯一』→ Accountable；『Controller 與 Processor 間必備文件』→ DPA。

常見陷阱：Controller 不能靠外包卸責；Owner(內部) vs Controller(法律) 別混；Responsible 可多人、Accountable 唯一。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：角色各有營運/法律責任；RACI 釐清權責（A 唯一）；共同決定目的＝Joint Controller、決定vs執行＝Controller/Processor 需 DPA；DPO 獨立監督。

Q1 . RACI 中哪個角色必須唯一？
→ Accountable（當責，不可分散/下授）。

Q2 . 兩家公司共同決定個資處理目的，是什麼關係？
→ Joint Controllers（共同控管者），需協議分責。

Q3 . Controller 能否藉由外包把責任完全轉給 Processor？
→ 不能；Controller 仍負選任監督等主要義務。
