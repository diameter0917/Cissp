考點：AAA 與因子分類

Exam: AAA & Factor Classification
01 — 考前重點濃縮

D5 後期。本封考鑑別因子分類與 AAA——把驗證方式歸到正確因子(Type 1/2/3)是 D5 最高頻考點。

Type 1 所知(密碼/PIN);Type 2 所有(Token/智慧卡/手機);Type 3 所是(生物特徵)。
MFA=跨不同因子類別;密碼+PIN 不是 MFA(都 Type 1)。
AAA:識別→鑑別→授權→可歸責;可歸責需唯一身分(禁共享帳號)。
02 — 因子速記
Type類別例子
Type 1所知 know密碼、PIN、安全問題
Type 2所有 haveToken、智慧卡、手機 OTP
Type 3所是 are指紋、臉、虹膜
03 — 關鍵名詞（中英）
Type 1/2/3（三因子類別）— 所知/所有/所是。
MFA（多因子鑑別）— 跨不同因子類別。
Accountability（可歸責）— 唯一身分＋日誌追溯。
OTP（一次性密碼）— Type 2(所有)。
Authentication（鑑別）— 證明聲稱身分。
04 — 常見陷阱

同類別兩因子不是 MFA(密碼+安全問題=兩個 Type 1)。

手機 OTP/智慧卡=Type 2(所有);生物特徵=Type 3。

可歸責需唯一身分;共享帳號破壞可歸責。

05 — 🎯 實戰考題
QUESTION 1

A system requires a password and a one-time code from a hardware token. These two factors represent which authentication types?

（某系統要求密碼加上硬體 Token 的一次性碼。這兩個因子代表哪些鑑別類型?）

A. Type 1 and Type 1 所知＋所知
B. Type 1 and Type 2 所知＋所有
C. Type 2 and Type 3 所有＋所是
D. Type 1 and Type 3 所知＋所是
✅ 解答與解析

正解：B。密碼＝Type 1(所知);硬體 Token 的 OTP＝Type 2(所有)。跨兩類別＝真正的 MFA。

為何其他選項錯：
  • A：兩個 Type 1 才是同類別(非此情況,且不算 MFA)。
  • C：沒有生物特徵(Type 3)。
  • D：Token 不是生物特徵。
QUESTION 2

Which combination does NOT qualify as multi-factor authentication?

（哪個組合『不』構成多因子鑑別?）

A. Password + fingerprint 密碼＋指紋
B. Password + security question 密碼＋安全問題
C. Smart card + PIN 智慧卡＋PIN
D. Fingerprint + OTP token 指紋＋OTP Token
✅ 解答與解析

正解：B。密碼與安全問題都是 Type 1(所知),屬同一類別→不是 MFA。MFA 須跨不同因子類別。

為何其他選項錯：
  • A：密碼(1)＋指紋(3),跨類別,是 MFA。
  • C：智慧卡(2)＋PIN(1),跨類別,是 MFA。
  • D：指紋(3)＋OTP(2),跨類別,是 MFA。
QUESTION 3

Which AAA component ensures actions can be traced back to a specific individual?

（AAA 中哪個元件確保行為可追溯到特定個人?）

A. Authentication 鑑別
B. Authorization 授權
C. Accountability 可歸責
D. Identification 識別
✅ 解答與解析

正解：C。Accountability(可歸責)透過唯一身分＋日誌,讓行為可追溯到個人;故須禁用共享帳號。

為何其他選項錯：
  • A：鑑別是證明身分,非事後追溯。
  • B：授權是決定能做什麼。
  • D：識別只是聲稱身分(輸入帳號)。
06 — (ISC)² 選最佳答案思路

因子歸類:密碼/PIN/問題=Type 1;Token/卡/OTP=Type 2;生物=Type 3。MFA 須跨類別。

可歸責=唯一身分＋日誌;看到『可追溯到個人』選 Accountability。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：Type 1 所知/2 所有/3 所是;MFA 須跨類別(密碼+安全問題不算);可歸責需唯一身分。

Q1 . 密碼＋安全問題算 MFA 嗎？
→ 不算;都是 Type 1(所知)。

Q2 . 智慧卡屬哪個因子類別？
→ Type 2(所有)。

Q3 . 確保行為可追溯到個人的 AAA 元件？
→ Accountability(可歸責)。
