考點：聯合身分協定

Exam: Federation Protocols
01 — 考前重點濃縮

本封考SAML vs OAuth vs OIDC 用途與 Kerberos 元件。釐清『授權 vs 鑑別』與票證流程。

OAuth=授權(非鑑別);OIDC=鑑別(OAuth 上發 ID token);SAML=企業 XML SSO。
Kerberos:KDC(AS+TGS)發 TGT→換 Service Ticket;時戳防重放(需時間同步)。
IdP 發身分、SP 信任消費。
02 — 協定速記
協定用途
SAML企業 Web SSO(XML)
OAuth 2.0授權(委派存取,非鑑別)
OIDC鑑別(OAuth 上,發 ID token)
Kerberos內網票證式 SSO
03 — 關鍵名詞（中英）
SAML（安全斷言標記語言）— 企業 XML SSO。
OAuth 2.0（開放授權）— 委派存取(非鑑別)。
OIDC（OpenID Connect）— OAuth 上的鑑別層。
Kerberos（票證認證）— KDC 發 TGT/服務票證。
IdP/SP（身分/服務提供者）— 發身分 vs 信任消費。
04 — 常見陷阱

OAuth 是授權不是鑑別;要登入(證明你是誰)用 OIDC。

Kerberos 需時間同步(時戳防重放)。

SAML(XML/企業) vs OIDC(JSON/行動)。

05 — 🎯 實戰考題
QUESTION 1

A mobile app lets a user grant it permission to access their Google Drive files WITHOUT sharing their Google password. Which protocol provides this delegated authorization?

（某行動 App 讓使用者授權它存取自己的 Google Drive 檔案,而『不分享 Google 密碼』。哪個協定提供此委派授權?）

A. SAML
B. OAuth 2.0
C. Kerberos
D. RADIUS
✅ 解答與解析

正解：B。OAuth 2.0 提供『委派授權』——讓 App 取得有限存取權(access token),而不交出密碼。

為何其他選項錯：
  • A：SAML 是企業 Web SSO 鑑別,非委派授權。
  • C：Kerberos 是內網票證認證。
  • D：RADIUS 是集中 AAA(撥入認證)。
QUESTION 2

Which protocol is built ON TOP of OAuth 2.0 specifically to provide authentication (verifying who the user is) via an ID token?

（哪個協定建構於 OAuth 2.0『之上』,專門透過 ID token 提供鑑別(確認使用者是誰)?）

A. SAML
B. OpenID Connect (OIDC)
C. LDAP
D. TACACS+
✅ 解答與解析

正解：B。OIDC 在 OAuth 2.0 之上加上鑑別層,發 ID token 證明使用者身分。用 OAuth 當登入是常見錯誤,應用 OIDC。

為何其他選項錯：
  • A：SAML 也做鑑別,但用 XML、非建於 OAuth 之上。
  • C：LDAP 是目錄查詢協定。
  • D：TACACS+ 是網路設備 AAA。
QUESTION 3

In Kerberos, what does a client first obtain after successfully authenticating, which is then used to request access to specific services?

（在 Kerberos 中,用戶端成功認證後首先取得什麼,之後用它來請求存取特定服務?）

A. Service Ticket 服務票證
B. Ticket Granting Ticket (TGT) 票證授予票證
C. Session cookie 會談 cookie
D. SAML assertion SAML 斷言
✅ 解答與解析

正解：B。Kerberos 流程:認證後先取得 TGT,再持 TGT 向 TGS 換取存取特定服務的 Service Ticket。

為何其他選項錯：
  • A：服務票證是『之後』用 TGT 換來的。
  • C：Kerberos 用票證非 cookie。
  • D：SAML 斷言屬 SAML 協定,非 Kerberos。
06 — (ISC)² 選最佳答案思路

授權→OAuth;鑑別→OIDC;企業 XML SSO→SAML;內網票證→Kerberos。

Kerberos:先 TGT 再 Service Ticket;需時間同步。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：OAuth 授權、OIDC 鑑別(OAuth 上)、SAML 企業 SSO、Kerberos 內網票證(TGT→服務票證,需時間同步)。

Q1 . 委派 App 存取資源而不交出密碼,用什麼協定？
→ OAuth 2.0。

Q2 . 建於 OAuth 上、提供鑑別發 ID token 的是？
→ OIDC。

Q3 . Kerberos 認證後先取得什麼？
→ TGT(票證授予票證),再換服務票證。
