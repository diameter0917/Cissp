考點：資料狀態與保護

Exam: Data States & Protection
01 — 考前重點濃縮

本封考資料三狀態的保護與 DLP 部署點。把每個狀態對應的控制背熟即可。

At Rest→儲存/磁碟加密;In Transit→TLS/IPsec/VPN;In Use→存取控制/機密運算(最難)。
DLP 部署:Network(出口,in transit)、Endpoint(端點,in use)、Storage/Discovery(掃描 at rest)。
02 — 狀態對應速記
狀態保護
At Rest 靜止儲存加密(AES)、存取控制
In Transit 傳輸TLS/IPsec/VPN
In Use 使用存取控制、機密運算(最難)
03 — 關鍵名詞（中英）
Data at Rest（靜止資料）— 儲存加密保護。
Data in Transit（傳輸中資料）— TLS/IPsec 保護。
Data in Use（使用中資料）— 記憶體中,最難保護。
DLP（資料外洩防護）— 網路/端點/儲存三部署。
Confidential Computing（機密運算）— 保護 in-use 資料。
04 — 常見陷阱

In Transit 用 TLS、At Rest 用儲存加密,別對調。

In Use 最難保護(資料須解密才能運算)。

只加密一種狀態不夠——三面都要顧。

05 — 🎯 實戰考題
QUESTION 1

Which control BEST protects data in transit between a web browser and a server?

（哪個控制最能保護瀏覽器與伺服器間『傳輸中』的資料?）

A. Full-disk encryption 全磁碟加密
B. TLS 傳輸層安全
C. Database access controls 資料庫存取控制
D. File hashing 檔案雜湊
✅ 解答與解析

正解：B。傳輸中(in transit)資料用 TLS(或 IPsec/VPN)加密通道保護。

為何其他選項錯：
  • A：全磁碟加密保護的是『靜止』資料(at rest),非傳輸。
  • C：存取控制保護儲存的資料,非傳輸通道。
  • D：雜湊驗完整性,不提供傳輸機密性。
QUESTION 2

An organization wants to detect and block sensitive files from being uploaded to personal cloud storage from employee laptops. Which DLP deployment is MOST appropriate?

（某組織想偵測並阻擋員工筆電將敏感檔上傳到個人雲端。最適合的 DLP 部署為何?）

A. Network DLP 網路 DLP
B. Endpoint DLP 端點 DLP
C. Storage/Discovery DLP 儲存掃描 DLP
D. Database encryption 資料庫加密
✅ 解答與解析

正解：B。要在端點管控本機操作(上傳/USB/列印)＝Endpoint DLP;它能在資料離開裝置前攔截。

為何其他選項錯：
  • A：Network DLP 在網路出口攔截,對加密的雲端上傳或離線端點較難掌握。
  • C：Storage DLP 掃描靜止資料庫存,非即時阻擋端點上傳。
  • D：資料庫加密保護儲存,非阻擋端點外傳。
QUESTION 3

Which data state is generally the MOST difficult to protect because the data must be decrypted to be processed?

（因為資料必須解密才能處理,哪個資料狀態通常最難保護?）

A. Data at rest 靜止
B. Data in transit 傳輸中
C. Data in use 使用中
D. Archived data 封存
✅ 解答與解析

正解：C。In Use(使用中)資料須在記憶體解密才能運算,故最難保護;機密運算(enclave)正是為補此落差。

為何其他選項錯：
  • A：靜止資料用儲存加密即可有效保護。
  • B：傳輸中用 TLS/IPsec 保護成熟。
  • D：封存屬於 at rest 的一種,可加密保護。
06 — (ISC)² 選最佳答案思路

狀態→控制:transit=TLS、rest=儲存加密、use=存取控制/機密運算(最難)。

DLP 部署:阻擋端點外傳=Endpoint;出口監控=Network;掃描靜止=Storage。對應資料狀態選部署點。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：三狀態:rest→儲存加密、transit→TLS、use→存取控制(最難);DLP 三部署對應三狀態。

Q1 . 保護傳輸中資料的最佳控制？
→ TLS(或 IPsec/VPN)。

Q2 . 阻擋端點上傳敏感檔用哪種 DLP？
→ Endpoint DLP(端點)。

Q3 . 哪個資料狀態最難保護、為什麼？
→ In use；須解密才能運算。
