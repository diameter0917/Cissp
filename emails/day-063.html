SAML/OAuth/OIDC 深入

SAML/OAuth/OIDC Deep Dive
01 — 中期 D5：聯合身分三大協定

今天深入聯合身分(federation)的三大協定：SAML（企業 SSO）、OAuth 2.0（授權）、OIDC（OAuth 之上的身分層）。釐清『誰負責鑑別、誰負責授權、token 怎麼流』，是 D5 高頻且易混的考點。

🗺️ 今天重點：SAML 斷言流程 → OAuth 授權碼流程 → OIDC → token 安全。
02 — 三者定位
協定用途格式
SAML 2.0企業 Web SSO(鑑別)XML 斷言
OAuth 2.0授權(委派存取，非鑑別)access token
OIDCOAuth 之上的鑑別層ID token(JWT)

💡 一句話：OAuth 是『授權』（給 App 存取你資源的權限），不是『鑑別』；要『證明你是誰』要用 OIDC（建於 OAuth 之上）。SAML 則是企業界的 SSO 老將。

03 — 角色與流程
SAML：使用者→SP(服務提供者)→重導到 IdP(身分提供者) 鑑別→IdP 發 SAML 斷言→SP 信任放行
OAuth 授權碼流程：使用者同意→授權伺服器發 code→App 用 code 換 access token→持 token 存取 API
為何用授權碼＋PKCE：避免 token 暴露於瀏覽器；PKCE 防授權碼攔截(行動/SPA 必備)
04 — Token 安全

JWT（JSON Web Token）常作 access/ID token：須驗簽章、驗發行者(iss)、受眾(aud)、過期(exp)。風險：token 竊取（replay）、過長壽命、未驗簽。對策：短壽命 access token＋refresh token、HTTPS、適當 scope、token 綁定。

05 — 關鍵名詞速查
SAML（安全斷言標記語言）— XML 斷言，企業 Web SSO。
OAuth 2.0（開放授權）— 委派存取的授權框架(非鑑別)。
OIDC（OpenID Connect）— OAuth 上的鑑別層，發 ID token。
IdP / SP（身分/服務提供者）— 發身分 vs 信任身分的一方。
Assertion（斷言）— IdP 對使用者身分/屬性的聲明。
Access/ID Token（存取/身分令牌）— 授權憑證 vs 身分憑證。
PKCE（授權碼交換證明）— 防授權碼攔截(行動/SPA)。
JWT（JSON Web Token）— 含簽章的 token 格式。
06 — 易混淆釐清

OAuth vs OIDC（最常考）：OAuth＝授權（App 能存取什麼資源）；OIDC＝鑑別（你是誰，發 ID token）。用 OAuth 當登入是常見錯誤，應用 OIDC。

SAML vs OIDC：兩者都能 SSO/鑑別。SAML 用 XML、偏企業 Web；OIDC 用 JSON/JWT、偏行動/API/現代 App。

IdP vs SP：IdP 發行身分（鑑別使用者）；SP 信任並消費身分（提供服務）。

07 — 條文與標準

SAML 2.0（OASIS）、OAuth 2.0＝RFC 6749、PKCE＝RFC 7636、OIDC（OpenID Foundation）、JWT＝RFC 7519。OAuth 2.1 草案整併最佳實務(強制 PKCE、棄用 implicit flow)。

08 — 真實事件：OAuth token 竊取
🔎 拿到 token，就拿到你的資料

OAuth token 竊取/濫用常見於：惡意第三方 App 誘導使用者授權過大 scope、或 token 經不安全管道(URL/log)外洩後被重放(replay)存取 API。也有攻擊濫用 OAuth 同意畫面（consent phishing）誘騙授權，繞過密碼與 MFA——因為拿的是『授權』而非『密碼』。

教訓：最小 scope、短壽命 token、嚴格 redirect URI 白名單、PKCE、監控異常授權；使用者要警覺第三方 App 的授權請求。token 等同鑰匙，需妥善保護與輪替。

📌 延伸：SSO 的單點失效與便利風險

SSO 提升便利與集中管控(一處停用即全斷)，但也是單點失效：IdP 一旦被攻破或當機，所有依賴它的服務都受影響。因此 IdP 需最高等級保護（MFA、監控、高可用）。便利與風險的權衡，是聯合身分的核心取捨。

09 — CISSP 考點提醒

選最佳答案：『委派 App 存取資源(授權)』→ OAuth；『證明使用者身分(鑑別)』→ OIDC；『企業 XML Web SSO』→ SAML；『行動 App 防授權碼攔截』→ PKCE。

常見陷阱：OAuth 是授權不是鑑別(登入用 OIDC)；IdP 發身分/SP 消費；token 要短命、最小 scope、驗簽章。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：SAML(XML 企業 SSO)、OAuth(授權非鑑別)、OIDC(OAuth 上的鑑別層發 ID token)；token 要短命/最小 scope/驗簽；IdP 發身分、SP 信任。

Q1 . OAuth 與 OIDC 的根本差別？
→ OAuth 是授權(存取資源)；OIDC 是鑑別(證明你是誰，發 ID token)。

Q2 . SAML 中發行身分斷言的一方叫什麼？
→ IdP(身分提供者)；SP(服務提供者)信任並消費。

Q3 . 行動/SPA 防止授權碼被攔截用什麼？
→ PKCE(授權碼交換證明)。
