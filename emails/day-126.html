QUESTION 1 · D1

An organization plans to adopt a SaaS provider that will process customer PII. Which action provides the BEST assurance of the provider's security posture before signing the contract?

（組織計畫採用一家將處理客戶個資的 SaaS 供應商。簽約前,哪個做法『最能』確保供應商的安全狀況?）

A. Ask the provider to complete a self-assessment questionnaire 請供應商填寫自評問卷
B. Run a penetration test against the provider's production environment 對供應商的正式環境執行滲透測試
C. Review the provider's independent third-party audit report and include a right-to-audit clause 審閱供應商的獨立第三方稽核報告,並在合約納入稽核權條款
D. Verify that the provider's marketing materials mention encryption 確認供應商行銷資料有提到加密
✅ 解答與解析

正解：C。供應鏈風險管理(SCRM)的核心是獨立、可驗證的保證(assurance):第三方稽核報告(如 SOC 2)由獨立稽核員出具,可信度遠高於自評;稽核權條款則確保簽約後仍有持續驗證的手段——事前調查+持續監督,正是 due diligence 的完整體現。

為何其他選項錯：
  • A：自評問卷是供應商自己說了算,缺乏獨立性,只能當輔助。
  • B：未經授權對他人環境滲透測試不合法也不可行,SaaS 多租戶環境通常禁止。
  • D：行銷資料不是保證機制,無稽核價值。
QUESTION 2 · D1

A quantitative risk analysis shows the annualized loss expectancy (ALE) of a threat is $20,000, while the only effective control costs $45,000 per year. What is the BEST course of action?

（定量風險分析顯示某威脅的年化損失期望值 ALE 為 2 萬美元,而唯一有效的控制措施每年要價 4.5 萬美元。『最佳』做法是什麼?）

A. Implement the control to eliminate the risk 導入該控制以消除風險
B. Formally accept the risk and document the decision 正式接受該風險並記錄決策
C. Ignore the risk since the control is too expensive 控制太貴,直接忽略該風險
D. Implement the control anyway to demonstrate due care 為展現 due care 仍導入該控制
✅ 解答與解析

正解：B。控制成本($45,000)大於風險本身($20,000),導入控制不符成本效益——風險導向的正確處置是風險接受(risk acceptance),且必須正式決策、書面記錄、由適當層級核准。接受是四種風險處置(降低/轉移/接受/迴避)中完全合法的一種,前提是「有意識地決定」。

為何其他選項錯：
  • A：花 4.5 萬防 2 萬的損失不符成本效益,且「消除風險」是絕對詞——風險無法歸零。
  • C：「忽略」與「接受」的差別在於是否經過分析、決策與記錄——忽略永遠是錯的。
  • D：due care 是採取「合理」的保護,明知不符成本效益仍硬上,反而不合理。
QUESTION 3 · D1

Senior management approved a strong security policy two years ago but has never reviewed compliance reports or followed up on audit findings since. Which concept has management MOST clearly failed to demonstrate?

（高階管理層兩年前核准了完善的安全政策,但此後從未檢視合規報告或追蹤稽核發現。管理層『最明顯』未能展現哪個概念?）

A. Due care 應盡注意
B. Separation of duties 職務分工
C. Least privilege 最小權限
D. Due diligence 應盡調查
✅ 解答與解析

正解：D。核准並建立政策是 due care(採取合理保護行動,do correct);持續檢視、監督、追蹤成效則是 due diligence(do detect)。題幹說管理層「做了政策」但「從不監督」——care 有做,diligence 沒做。這是 Day 125 陷阱表裡「due care/diligence 對調」的標準考法。

為何其他選項錯：
  • A：due care 是建立合理保護——政策已核准,這部分有做到。
  • B：職務分工是防止單人舞弊的控制設計,與題幹的「不監督」無關。
  • C：最小權限是存取控制原則,與管理層的治理義務無關。
QUESTION 4 · D2

A new customer analytics dataset has been created. Who is PRIMARILY responsible for determining its classification level?

（一份新的客戶分析資料集剛建立。誰『主要』負責決定它的分類等級?）

A. The data owner 資料擁有者
B. The data custodian 資料保管者
C. The security administrator 資安管理員
D. The end users of the dataset 資料集的終端使用者
✅ 解答與解析

正解：A。資料擁有者(data owner)——通常是對該業務資料當責的管理階層——負責決定分類等級、核准存取規則;保管者(custodian)依 owner 的決定執行備份、加密等日常保護。記憶法:「owner 決定,custodian 執行,user 遵守」。

為何其他選項錯：
  • B：custodian 負責依分類實施技術保護,不決定分類。
  • C：資安管理員提供框架與工具支援,分類決定權在業務端 owner。
  • D：使用者只需依標示遵守處理規則,無決定權。
QUESTION 5 · D2

A batch of solid-state drives (SSDs) containing highly confidential data is being decommissioned and will leave the organization's control. Which sanitization method provides the MOST reliable protection?

（一批含高機密資料的 SSD 即將除役並脫離組織控制。哪種銷毀方法提供『最可靠』的保護?）

A. Physical destruction of the drives 實體銷毀硬碟
B. Multiple-pass overwriting of all sectors 對所有磁區多次覆寫
C. Degaussing the drives 對硬碟消磁
D. Performing a full format of each drive 對每顆硬碟完整格式化
✅ 解答與解析

正解：A。兩個關鍵條件:SSD + 脫離組織控制 + 高機密。SSD 的磨損平均(wear leveling)與備用區塊使覆寫無法保證觸及所有實體儲存單元;消磁對快閃記憶體完全無效。媒體將離開控制範圍時,最高保證等級就是實體銷毀(destroy)。

為何其他選項錯：
  • B：覆寫在 SSD 上因 wear leveling 無法覆蓋所有區塊,保證不足。
  • C：消磁只對磁性媒體有效,SSD 是快閃記憶體,消磁無效。
  • D：格式化只重建檔案系統結構,資料可輕易還原,連 clearing 都稱不上。
QUESTION 6 · D3

A military system must ensure that users cannot read documents classified above their clearance level and cannot write information down to lower classification levels. Which security model BEST enforces these requirements?

（一套軍事系統必須確保使用者不能讀取高於其許可等級的文件,也不能把資訊寫入較低的機密等級。哪個安全模型『最能』落實這些要求?）

A. Bell-LaPadula 貝爾-拉帕杜拉模型
B. Biba 畢巴模型
C. Clark-Wilson 克拉克-威爾遜模型
D. Brewer-Nash 布魯爾-納許模型
✅ 解答與解析

正解：A。題幹描述的正是 Bell-LaPadula 的兩條規則:simple security property(no read up)+star property(no write down)——為機密性而生的模型,軍事情境是它的招牌。記憶法:「BLP 保密:不上讀、不下寫」。

為何其他選項錯：
  • B：Biba 保完整性,規則相反(no read down / no write up)。
  • C：Clark-Wilson 用良構交易與職務分工保商業完整性,無分級讀寫規則。
  • D：Brewer-Nash(Chinese Wall)防利益衝突,依存取歷史動態限制,與分級無關。
QUESTION 7 · D3

An organization must send contracts to many external partners with both confidentiality and non-repudiation of origin. Which approach BEST meets both requirements?

（組織須將合約傳送給多個外部夥伴,同時要求機密性與來源不可否認性。哪種做法『最能』同時滿足兩者?）

A. Sign with the sender's private key, then encrypt with each recipient's public key 用寄件者私鑰簽章,再用各收件者公鑰加密
B. Encrypt with a symmetric key shared among all partners 用所有夥伴共享的對稱金鑰加密
C. Attach a hash of the contract to the plaintext message 在明文訊息附上合約的雜湊值
D. Use a message authentication code (MAC) with a shared secret 使用共享密鑰的訊息鑑別碼 MAC
✅ 解答與解析

正解：A。拆需求:機密性→用收件者公鑰加密(只有其私鑰能解);來源不可否認→用寄件者私鑰簽章(只有寄件者能產生,第三方可驗證)。兩者疊加就是標準的「簽章+加密」混合流程。記憶法:「加密用對方的公鑰,簽名用自己的私鑰」。

為何其他選項錯：
  • B：共享對稱金鑰有機密性但無不可否認——任何持鑰者都可能是來源。
  • C：明文附雜湊只驗完整性,無機密性也無來源證明,且雜湊可被連同訊息一起替換。
  • D：MAC 提供完整性與驗證,但共享密鑰使雙方都能產生,無法對第三方證明來源。
QUESTION 8 · D3

A company is selecting a fire suppression system for a data center where operations staff are regularly present. Which system is MOST appropriate?

（公司要為一座維運人員經常在場的資料中心選擇滅火系統。哪種系統『最』合適?）

A. Wet-pipe water sprinkler system 濕式水噴灑系統
B. Clean-agent gas suppression system (e.g., FM-200) 潔淨氣體滅火系統(如 FM-200)
C. Carbon dioxide (CO2) flooding system 二氧化碳全區放射系統
D. Dry-powder extinguishing system 乾粉滅火系統
✅ 解答與解析

正解：B。兩個限制條件:有人在場(排除危及人命的選項)+電子設備(排除毀損設備的選項)。潔淨氣體(FM-200/HFC-227ea、Novec 等)在設計濃度下對人體相對安全、不導電、不留殘留物,是有人資料中心的標準答案——再次呼應「人命 > 資產」。

為何其他選項錯：
  • A：水對人安全但會毀掉帶電設備,是最後手段而非首選。
  • C：CO2 靠排擠氧氣滅火,對在場人員有窒息致命風險,適合無人空間。
  • D：乾粉殘留物具腐蝕性,會嚴重損壞電子設備,不適合資料中心。
QUESTION 9 · D4

A network team detects that an attacker on the local network is intercepting traffic between workstations and the default gateway by sending forged ARP replies. Which control BEST mitigates this attack?

（網路團隊偵測到內網攻擊者透過偽造 ARP 回應,攔截工作站與預設閘道之間的流量。哪個控制『最能』緩解此攻擊?）

A. Deploy a signature-based NIDS at the perimeter 在邊界部署特徵式 NIDS
B. Enable dynamic ARP inspection with DHCP snooping on switches 在交換器啟用動態 ARP 檢測搭配 DHCP snooping
C. Increase the frequency of vulnerability scans 提高弱點掃描頻率
D. Require complex passwords on all workstations 要求所有工作站使用複雜密碼
✅ 解答與解析

正解：B。題幹是教科書式的 ARP spoofing / on-path(MITM)攻擊。動態 ARP 檢測(DAI)以 DHCP snooping 建立的 IP-MAC 綁定表驗證每個 ARP 封包,直接在攻擊發生的第二層阻斷偽造回應——對症下藥的預防性控制。

為何其他選項錯：
  • A：邊界 NIDS 看不到內網交換的第二層流量,且偵測不等於阻止。
  • C：弱點掃描找設定與軟體弱點,對進行中的 ARP 攻擊無緩解作用。
  • D：密碼強度與第二層流量攔截完全無關。
QUESTION 10 · D4

Two branch offices need a permanent site-to-site tunnel over the Internet that provides confidentiality, integrity, and origin authentication for all traffic between them. Which solution BEST meets the requirement?

（兩個分公司需要一條經網際網路的常設站對站通道,為兩地間所有流量提供機密性、完整性與來源鑑別。哪個方案『最』符合需求?）

A. A GRE tunnel between the two gateways 兩閘道間建立 GRE 通道
B. PPTP with MS-CHAPv2 authentication 使用 MS-CHAPv2 驗證的 PPTP
C. A TLS portal for users to access applications 供使用者存取應用的 TLS 入口網站
D. An IPsec VPN using ESP in tunnel mode 使用 ESP 通道模式的 IPsec VPN
✅ 解答與解析

正解：D。需求關鍵:站對站、所有流量、機密+完整+來源鑑別。IPsec ESP 同時提供加密、完整性與鑑別;tunnel mode 封裝整個原始 IP 封包,正是 gateway-to-gateway 的標準架構,對兩地所有流量透明生效。

為何其他選項錯：
  • A：GRE 只做封裝,本身不加密、不驗證,三項需求全不滿足。
  • B：PPTP/MS-CHAPv2 有已知重大弱點,早已不建議用於安全通道。
  • C：TLS 入口保護的是使用者到應用的個別連線,不是兩站間全流量的常設通道。
QUESTION 11 · D4

After a ransomware infection spread rapidly from one workstation to servers across the entire flat network, management asks how to limit lateral movement in the future. Which architectural change BEST addresses this?

（勒索軟體從一台工作站迅速擴散到整個扁平網路上的伺服器後,管理層詢問未來如何限制橫向移動。哪個架構性變更『最能』解決此問題?）

A. Deploy a stronger antivirus product on all endpoints 在所有端點部署更強的防毒產品
B. Increase the frequency of full backups 提高完整備份的頻率
C. Add an additional perimeter firewall 增加一道邊界防火牆
D. Segment the network into zones with controlled traffic between them 將網路分割為多個區段,並管制區段間流量
✅ 解答與解析

正解：D。題幹點名兩個關鍵:flat network(擴散的根因)與限制橫向移動(目標),又指定要架構性(architectural)變更——答案就是網路分段/微分段:把網路切成受控區域,區段間流量須經過濾,單點淪陷不再等於全網淪陷。

為何其他選項錯：
  • A：防毒是端點控制,不是架構變更,對零時差勒索軟體的橫向移動幫助有限。
  • B：備份改善復原能力,不能限制擴散——答非所問。
  • C：邊界防火牆管內外流量,對「內網之間」的橫向移動無能為力。
QUESTION 12 · D5

A retail company with high staff turnover wants to assign permissions based on job functions so that access changes automatically follow position changes. Which access control model BEST fits?

（一家人員流動率高的零售公司,希望依職務功能指派權限,讓存取權隨職位異動自動跟著改變。哪種存取控制模型『最』適合?）

A. Discretionary access control (DAC) 自主式存取控制
B. Mandatory access control (MAC) 強制式存取控制
C. Role-based access control (RBAC) 角色基礎存取控制
D. Rule-based access control 規則基礎存取控制
✅ 解答與解析

正解：C。題幹的訊號詞:job functions(職務功能)+高流動率。RBAC 把權限綁在「角色」上,人員到職/異動只要改變角色指派,權限自動跟上——大幅降低管理負擔,也天然抑制權限累積。「依職務給權限」幾乎等於 RBAC 的定義。

為何其他選項錯：
  • A：DAC 由資源擁有者個別授權,人員異動時要逐一調整,高流動率下是災難。
  • B：MAC 依安全標籤與許可等級,適合軍政高機密環境,對零售業過度且僵硬。
  • D：規則基礎依全域規則(如防火牆 ACL)判斷,與職務功能無對應關係。
QUESTION 13 · D5

A company wants employees of its business partner to access a shared cloud application using the partner's own corporate credentials, without creating local accounts. Which solution BEST achieves this?

（公司希望商業夥伴的員工用夥伴自家的企業帳號存取共用雲端應用,而不必另建本地帳號。哪個方案『最能』達成?）

A. Create shared guest accounts for partner staff 為夥伴員工建立共用訪客帳號
B. Replicate the partner's LDAP directory into the local domain 把夥伴的 LDAP 目錄複寫到本地網域
C. Store partner passwords in an enterprise password vault 將夥伴密碼存放於企業密碼保險庫
D. Implement federated identity using SAML 以 SAML 實作聯邦式身分
✅ 解答與解析

正解：D。「用自家帳號存取他方服務、不建本地帳號」就是聯邦式身分(federation)的定義:夥伴的 IdP(身分提供者)驗證使用者後,以 SAML assertion 向服務方(SP)證明身分——信任跨組織傳遞,帳號生命週期留在原組織管理。

為何其他選項錯：
  • A：共用帳號摧毀可歸責性(accountability),是反模式。
  • B：目錄複寫等於在本地重建帳號,管理負擔與同步風險俱增,正是題目要避免的。
  • C：集中存他組織的密碼既增風險又不消除本地帳號需求,完全不對題。
QUESTION 14 · D5

An internal audit finds that several employees who transferred between departments still hold permissions from their previous roles. Which control BEST prevents this situation from recurring?

（內部稽核發現多位跨部門調動的員工仍保有前職務的權限。哪個控制『最能』防止此情況再次發生?）

A. Enforce a stronger password policy 強化密碼原則
B. Conduct periodic access reviews tied to the transfer process 實施與調動流程掛鉤的定期存取權審查
C. Require multifactor authentication for all systems 全系統要求多因子驗證
D. Log all privileged account activity 記錄所有特權帳號活動
✅ 解答與解析

正解：B。題幹描述的是權限蔓延(privilege creep / authorization creep)——身分生命週期管理的失效。對症的控制是定期存取權審查/再認證(access review & recertification),並在調動(transfer)流程觸發權限重新評估,移除不再需要的舊權限。

為何其他選項錯：
  • A：密碼強度管「驗證」,問題出在「授權」殘留,面向不對。
  • C：MFA 同樣是驗證控制,擋不住合法帳號持有多餘權限。
  • D：記錄是偵測性控制,能發現濫用但不能防止權限殘留本身。
QUESTION 15 · D6

A consulting firm has been hired to perform a penetration test on a client's production systems. What MUST be obtained FIRST, before any testing activity begins?

（一家顧問公司受聘對客戶的正式系統執行滲透測試。任何測試活動開始前,『必須首先』取得什麼?）

A. A list of known vulnerabilities from the client's last scan 客戶上次掃描的已知弱點清單
B. Administrative credentials for the target systems 目標系統的管理者帳號
C. The client's network topology diagrams 客戶的網路拓撲圖
D. Signed management authorization defining the scope and rules of engagement 由管理層簽署、界定範圍與交戰規則的書面授權
✅ 解答與解析

正解：D。沒有書面授權的滲透測試就是非法入侵。開始任何測試前,必須先取得管理層簽署的授權書(俗稱 get-out-of-jail-free letter),明訂範圍、時間窗、允許手法、緊急聯絡與中止條件(rules of engagement)。法律保障先於一切技術準備。

為何其他選項錯：
  • A：弱點清單是測試的參考輸入,沒有授權時連看的立場都沒有。
  • B：是否提供帳號取決於測試型態(黑箱不給),且絕非「必須首先」。
  • C：拓撲圖同為選擇性輸入,授權未定前不應交付。
QUESTION 16 · D6

A cloud provider's enterprise customers demand evidence that its security controls operated effectively over the past 12 months. Which report BEST satisfies this demand?

（雲端供應商的企業客戶要求證明其安全控制在過去 12 個月有效運作。哪份報告『最能』滿足此要求?）

A. SOC 2 Type II report SOC 2 第二型報告
B. SOC 2 Type I report SOC 2 第一型報告
C. SOC 1 report SOC 1 報告
D. SOC 3 general-use report SOC 3 通用報告
✅ 解答與解析

正解：A。兩個關鍵:安全控制(→SOC 2,信任服務準則)與一段期間的運作有效性(→Type II,涵蓋 6–12 個月的運作測試)。記憶法:「Type I 看某一天的設計,Type II 看一段期間的運作」;「SOC 1 管財報,SOC 2 管安全,SOC 3 是對外摘要」。

為何其他選項錯：
  • B：Type I 只評估「某個時間點」的控制設計,不驗證期間運作有效性。
  • C：SOC 1 針對影響客戶「財務報告」的控制,不是安全控制。
  • D：SOC 3 是可公開的摘要版,缺乏測試細節,無法滿足企業客戶的查證需求。
QUESTION 17 · D7

An organization wants to verify that its disaster recovery site can actually run production workloads, but cannot tolerate any interruption to the primary site's operations. Which DRP test type is MOST appropriate?

（組織想驗證災難復原站點能真正承載正式工作負載,但完全不能容忍主站營運中斷。哪種 DRP 測試類型『最』合適?）

A. Tabletop (structured walk-through) exercise 桌上(結構化演練)測試
B. Checklist review 檢查表審查
C. Parallel test 並行測試
D. Full-interruption test 完全中斷測試
✅ 解答與解析

正解：C。兩個條件:要實際跑起來驗證(排除紙上作業)+主站不能中斷(排除切換)。並行測試(parallel test)讓復原站點實際啟動並處理(複製的)工作負載,主站同時照常營運——「不中斷前提下最徹底」的測試,正好卡在兩個條件的交集。

為何其他選項錯：
  • A：桌上演練只是討論情境,無法驗證站點「真的能跑」。
  • B：檢查表審查是最低強度的文件核對,更不涉及實機。
  • D：完全中斷測試要把主站切掉,直接違反「不容忍中斷」的前提。
QUESTION 18 · D7

A SOC analyst confirms that a file server is actively encrypting files due to ransomware. According to the incident response process, what should the team do NEXT?

（SOC 分析師確認一台檔案伺服器正因勒索軟體而持續加密檔案。依事件回應流程,團隊『接下來』應該做什麼?）

A. Rebuild the server and restore data from backups 重建伺服器並從備份還原資料
B. Wipe the malware from the server immediately 立即清除伺服器上的惡意程式
C. Isolate the affected server to contain the spread 隔離受影響伺服器以圍堵擴散
D. Conduct a lessons-learned meeting with stakeholders 與利害關係人召開檢討會議
✅ 解答與解析

正解：C。NEXT 題先定位:偵測與確認(detection/analysis)已完成,事件回應流程的下一站是圍堵(containment)——隔離受害主機,阻止災情擴大,同時保全證據。順序:準備→偵測→圍堵→根除→復原→檢討。

為何其他選項錯：
  • A：復原(recovery)在根除之後,未圍堵就還原只會再次感染。
  • B：根除(eradication)在圍堵之後,立刻清除還會破壞鑑識證據。
  • D：檢討(lessons learned)是整個流程的最後一步。
QUESTION 19 · D8

After a critical vulnerability was disclosed in a widely used open-source library, a company spent weeks manually determining which of its applications were affected. Which practice would BEST enable rapid identification in the future?

（某廣泛使用的開源函式庫爆出重大弱點後,公司花了數週人工清查哪些應用受影響。哪個做法『最能』讓未來得以快速識別?）

A. Schedule annual penetration tests for all applications 為所有應用安排年度滲透測試
B. Deploy a web application firewall in front of all services 在所有服務前部署 WAF
C. Maintain a software bill of materials (SBOM) for every application 為每個應用維護軟體物料清單 SBOM
D. Require manual code review for all new releases 所有新版本一律人工程式碼審查
✅ 解答與解析

正解：C。這是 Log4Shell 式情境。SBOM(software bill of materials)逐一記錄每個應用包含的元件與版本——弱點公告一出,查詢 SBOM 就能在數小時內列出所有受影響系統,取代數週的人工盤點。這正是軟體供應鏈安全的核心工件。

為何其他選項錯：
  • A：年度滲透測試頻率太低,且測的是可利用性,不是元件清查。
  • B：WAF 是緩解層,無法告訴你「哪些應用內含該函式庫」。
  • D：人工審查看自寫程式碼品質,對第三方相依元件的追蹤幫助有限且極慢。
QUESTION 20 · D8

An audit reveals that developers can push code changes directly into the production environment. Which control BEST addresses the risk created by this practice?

（稽核發現開發人員可以把程式碼變更直接推進正式環境。哪個控制『最能』處理這種做法造成的風險?）

A. Provide developers with additional secure coding training 加強開發人員的安全編碼訓練
B. Enforce change management with separation of duties, requiring independent review and approval before deployment 落實變更管理與職務分工,部署前須經獨立審查與核准
C. Scan the production environment for vulnerabilities weekly 每週對正式環境做弱點掃描
D. Encrypt the source code repository 為程式碼儲存庫加密
✅ 解答與解析

正解：B。開發者直推正式環境的根本問題是缺乏職務分工與變更控制——一個人就能引入(惡意或錯誤的)變更且無人把關。對症的是流程控制:變更管理+獨立審查核准+開發/部署權限分離。工具與訓練都好,但「流程 > 工具」,先把控制結構補上。

為何其他選項錯：
  • A：訓練降低無心之過,對故意繞過或單人全權的結構性風險無效。
  • C：每週掃描是事後偵測,變更當下的風險窗口依然敞開。
  • D：儲存庫加密保護的是原始碼機密性,與「誰能改正式環境」無關。
