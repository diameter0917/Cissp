考點：防火牆與 IDS

Exam: Firewalls & IDS
01 — 考前重點濃縮

本封考防火牆類型與 IDS vs IPS、誤報/漏報。判斷情境該用哪種設備。

IDS 旁路告警 / IPS 在線阻擋;WAF 護 Web(L7)。
誤報(false positive)=正常被當攻擊;漏報(false negative)=攻擊沒抓到(更危險)。
防火牆預設應 default deny;狀態防火牆追蹤連線。
02 — 核心速記
設備/概念重點
IDS旁路、偵測告警
IPS在線、偵測阻擋
WAFL7 護 Web(SQLi/XSS)
False Negative漏報,最危險
03 — 關鍵名詞（中英）
IDS / IPS（入侵偵測/防禦）— 告警 vs 阻擋。
Stateful Firewall（狀態防火牆）— 追蹤連線狀態。
WAF（Web 應用防火牆）— L7 護 Web。
False Positive/Negative（誤報/漏報）— 正常誤判/攻擊漏抓。
Default Deny（預設拒絕）— 未明確允許即拒絕。
04 — 常見陷阱

IDS(告警) vs IPS(阻擋);IPS 誤報會擋到正常流量。

漏報比誤報危險(真攻擊溜過去)。

WAF(L7) vs 傳統防火牆(L3/L4)。

05 — 🎯 實戰考題
QUESTION 1

A device sits inline and actively blocks detected malicious traffic in real time. This is a(n):

（某設備在線部署,即時主動阻擋偵測到的惡意流量。這是:）

A. IDS 入侵偵測系統
B. IPS 入侵防禦系統
C. Passive sniffer 被動側錄器
D. Syslog server 日誌伺服器
✅ 解答與解析

正解：B。在線＋主動阻擋＝IPS;IDS 是旁路、只告警不阻擋。

為何其他選項錯：
  • A：IDS 旁路被動、只告警。
  • C：側錄器只擷取流量,不阻擋。
  • D：日誌伺服器收集日誌,非阻擋設備。
QUESTION 2

An IDS fails to alert on an actual intrusion. This error is called a:

（某 IDS 對一次真實入侵未發出告警。這種錯誤稱為:）

A. False positive 誤報
B. False negative 漏報
C. True positive 真陽性
D. Clipping level 裁切門檻
✅ 解答與解析

正解：B。真實攻擊未被偵測＝False Negative(漏報),是最危險的錯誤類型。

為何其他選項錯：
  • A：誤報是把正常當攻擊(發了不該發的告警)。
  • C：真陽性是正確偵測到攻擊。
  • D：裁切門檻是觸發告警的閾值設定。
QUESTION 3

Which control is specifically designed to inspect HTTP traffic and block application-layer attacks like SQL injection and XSS?

（哪個控制專門檢視 HTTP 流量並阻擋 SQL 注入、XSS 等應用層攻擊?）

A. Packet-filter firewall 封包過濾防火牆
B. Stateful firewall 狀態防火牆
C. Web Application Firewall (WAF)
D. Network IDS
✅ 解答與解析

正解：C。WAF 在 L7 檢視 HTTP 內容,專擋 SQLi/XSS 等應用層攻擊。

為何其他選項錯：
  • A／B：封包/狀態防火牆在 L3/L4,看不到應用層內容。
  • D：NIDS 偵測告警,非專門阻擋應用層攻擊的內容過濾。
06 — (ISC)² 選最佳答案思路

在線阻擋=IPS;旁路告警=IDS;L7 護 Web=WAF。

漏報(false negative)最危險;防火牆 default deny。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：IDS 告警/IPS 阻擋、WAF 護 Web(L7);漏報最危險;防火牆預設拒絕。

Q1 . 在線即時阻擋惡意流量的設備？
→ IPS(入侵防禦系統)。

Q2 . 真實攻擊未被偵測叫什麼、為何危險？
→ 漏報(false negative);真攻擊溜過去。

Q3 . 專擋 SQLi/XSS 的應用層控制？
→ WAF(Web 應用防火牆)。
