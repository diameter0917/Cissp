OWASP 與常見漏洞

OWASP & Common Vulns
01 — OWASP 與常見 Web 漏洞

Web 應用是最大攻擊面。OWASP Top 10 是業界公認的高風險清單。今天輪廓化最常考的 注入（Injection）、XSS、CSRF 與其防禦原則：輸入驗證、輸出編碼、參數化。

🗺️ 今天重點：OWASP Top 10 概念 → SQL 注入 → XSS → CSRF → 防禦原則。
02 — 三大常見漏洞
漏洞本質防禦
Injection（SQLi/命令）把惡意輸入當指令執行參數化查詢、輸入驗證、最小權限
XSS在頁面注入惡意腳本，於他人瀏覽器執行輸出編碼、CSP、輸入驗證
CSRF誘使已登入使用者送出非本意請求CSRF token、SameSite cookie

💡 一句話：注入＝信任了不該信的輸入；XSS＝沒編碼輸出；CSRF＝沒驗證請求意圖。三者根因都是『信任邊界沒守好』。

03 — 防禦黃金原則
不信任任何輸入：白名單驗證（validate）優於黑名單
參數化/預備語句：讓資料永遠是資料、不變成指令（擋注入）
輸出編碼（output encoding）：按情境（HTML/JS/URL）編碼（擋 XSS）
最小權限：資料庫帳號權限最小化，降低被注入後的衝擊
04 — 關鍵名詞速查
OWASP Top 10（十大 Web 風險）— 業界公認的高風險漏洞清單。
SQL Injection（SQL 注入）— 惡意輸入被當 SQL 執行，竊取/竄改資料。
XSS（跨站腳本）— 在頁面注入腳本於他人瀏覽器執行。
CSRF（跨站請求偽造）— 誘使已登入者送出非本意請求。
Input Validation（輸入驗證）— 以白名單檢核輸入合法性。
Output Encoding（輸出編碼）— 依情境編碼輸出以中和腳本。
05 — 易混淆釐清

XSS vs CSRF：XSS 是『在受害者瀏覽器執行攻擊者的腳本』（利用網站對使用者的信任輸出）；CSRF 是『冒用使用者身分送請求』（利用網站對使用者的信任）。防法不同：XSS 靠輸出編碼/CSP，CSRF 靠 token/SameSite。

驗證 vs 編碼：輸入驗證擋不合法輸入；輸出編碼讓資料在呈現時不被當程式。兩者都要、不可互相取代。

06 — 條文與標準

OWASP Top 10 與 OWASP ASVS（應用安全驗證標準）是 Web 安全基準；CWE 分類弱點類型。安全編碼指引常引用 OWASP Cheat Sheets。

07 — 真實事件：SQL 注入外洩
🔎 一個輸入框，撈走整個資料庫

SQL 注入是歷史最久、危害最大的漏洞之一：攻擊者在登入框或查詢參數輸入特製字串，讓後端把它當 SQL 執行，進而繞過登入或匯出整個資料庫。多起大型外洩（含知名企業）都肇因於未參數化的查詢。

教訓：參數化查詢＋輸入驗證＋資料庫最小權限幾乎可根除 SQLi。這是安全編碼最基本也最關鍵的一課。

08 — CISSP 考點提醒

選最佳答案：『防 SQL 注入最有效』→ 參數化查詢（prepared statements）；『防 XSS』→ 輸出編碼＋CSP；『防 CSRF』→ CSRF token／SameSite cookie。

常見陷阱：單靠黑名單過濾不可靠（用白名單）；XSS 與 CSRF 防法別搞混。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：OWASP Top 10 列高風險；注入靠參數化、XSS 靠輸出編碼、CSRF 靠 token；根因都是信任邊界沒守好。

Q1 . 防 SQL 注入最根本的做法？
→ 參數化查詢/預備語句（讓資料不被當指令）。

Q2 . XSS 與 CSRF 的防禦分別是？
→ XSS：輸出編碼/CSP；CSRF：CSRF token/SameSite cookie。

Q3 . 輸入驗證該用白名單還黑名單？
→ 白名單（只允許已知合法），較黑名單可靠。

Broken Access Control、Security Misconfiguration、Software Supply Chain Failures、Cryptographic Failures、Injection、Insecure Design、Authentication Failures、Integrity Failures、Logging/Alerting Failures、Mishandling Exceptions。

▍常見注入類漏洞

SQLi 注入（→參數化查詢）、XSS 跨站腳本（→輸出編碼）、CSRF 跨站請求偽造（→anti-CSRF token）。Design flaw 設計缺陷（做對也有漏）vs Implementation flaw 實作缺陷。

對應官方 §8.5.1 原始碼層級弱點（OWASP）。

§8.5.1 Security weaknesses and vulnerabilities at the source-code level
• A source code vulnerability is a code defect providing a threat actor with an opportunity to compromise the security of a software system
• source code vulns are caused by design or implementation flaws
• design flaw: if dev did everything correctly, there would still be a vulnerability
• implementation flaw: dev incorrectly implemented part of a good design
• the OWASP top 10 vulnerabilities (2025 edition):
• Broken access control
• Security misconfiguration
• Software supply chain failures
• Cryptographic failures
• Injection
• Insecure design
• Authentication failures
• Software or data integrity failures
• Security logging and alerting failures
• Mishandling of exceptional conditions
