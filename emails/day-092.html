考點：對稱 vs 非對稱選用

Exam: Symmetric vs Asymmetric Selection
01 — 考前重點濃縮

本封考對稱 vs 非對稱的選用、金鑰數量計算、加密/簽章的鑰方向。情境題判斷『該用哪種、用誰的鑰』。

對稱(AES)快→加密大量資料;非對稱(RSA/ECC)慢→金鑰交換與簽章。
加密用對方公鑰、簽章用自己私鑰。
對稱金鑰數＝n(n-1)/2;混合加密(TLS)兼得。
02 — 核心速記
需求用什麼
加密大量資料對稱(AES)
安全交換金鑰非對稱/DH
確保只有某人能解用對方公鑰加密
不可否認(簽章)用自己私鑰簽
03 — 關鍵名詞（中英）
Symmetric（對稱加密）— 單鑰、快、加大量資料。
Asymmetric（非對稱加密）— 公私鑰、慢、換鑰/簽章。
Hybrid Encryption（混合加密）— 非對稱換鑰+對稱加資料(TLS)。
Digital Signature（數位簽章）— 私鑰簽、公鑰驗。
Key Pair（金鑰對）— 公鑰/私鑰。
04 — 常見陷阱

加密用對方公鑰、簽章用自己私鑰(方向別反)。

對稱快、非對稱慢:大量資料一定用對稱。

ECC 同強度金鑰更短(適行動/IoT)。

05 — 🎯 實戰考題
QUESTION 1

Alice wants to send a confidential message that only Bob can read. Which key should she use to encrypt it?

（Alice 想送一則只有 Bob 能讀的機密訊息。她該用哪把鑰加密?）

A. Alice's private key Alice 的私鑰
B. Alice's public key Alice 的公鑰
C. Bob's public key Bob 的公鑰
D. Bob's private key Bob 的私鑰
✅ 解答與解析

正解：C。要確保只有 Bob 能解,用 Bob 的公鑰加密——只有 Bob 的私鑰能解開(機密性)。

為何其他選項錯：
  • A：用 Alice 私鑰是『簽章』用途,任何人用她公鑰都能解,不保機密。
  • B：用 Alice 自己的公鑰,只有 Alice 能解,Bob 解不開。
  • D：別人無法取得 Bob 的私鑰來加密。
QUESTION 2

Which is the BEST reason TLS uses asymmetric encryption to exchange a symmetric session key, rather than using asymmetric encryption for all data?

（為何 TLS 用非對稱加密交換一把對稱會談金鑰,而非全程用非對稱加密所有資料? 最佳理由是:）

A. Asymmetric is more secure 非對稱更安全
B. Symmetric encryption is much faster for bulk data 對稱加密大量資料快得多
C. Symmetric keys never expire 對稱金鑰永不過期
D. Asymmetric cannot encrypt data 非對稱無法加密資料
✅ 解答與解析

正解：B。非對稱運算昂貴、慢,不適合大量資料;故用非對稱安全交換一把對稱會談金鑰,再用對稱(快)加密大量資料——混合加密兼得安全與效能。

為何其他選項錯：
  • A：兩者安全性取決於正確使用,非『非對稱一定更安全』;選此誤解了原因。
  • C：對稱金鑰也應輪替,非永不過期。
  • D：非對稱可以加密資料,只是慢、有大小限制。
QUESTION 3

In a group of 10 people who all need to communicate securely using ONLY symmetric encryption, how many unique keys are required?

（10 人全部需用『純對稱加密』兩兩安全通訊,需要幾把獨特金鑰?）

A. 10
B. 20
C. 45
D. 100
✅ 解答與解析

正解：C。對稱兩兩通訊金鑰數＝n(n-1)/2 = 10×9/2 = 45。這也說明對稱金鑰分發隨人數快速膨脹的問題。

為何其他選項錯：
  • A：10 是人數,非金鑰數。
  • B：20 為計算錯誤。
  • D：100 是 n² ,未除以 2 且含自身配對。
06 — (ISC)² 選最佳答案思路

選鑰:機密性→對方公鑰加密;不可否認→自己私鑰簽。大量資料→對稱;換鑰→非對稱。

金鑰數 n(n-1)/2;混合加密理由＝對稱對大量資料快。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：對稱快(加資料)、非對稱慢(換鑰/簽章);加密用對方公鑰、簽章用自己私鑰;對稱金鑰 n(n-1)/2。

Q1 . 要讓只有 Bob 能解密,用誰的鑰加密？
→ Bob 的公鑰。

Q2 . TLS 用混合加密的最佳理由？
→ 對稱加密大量資料快得多;用非對稱只為安全交換對稱金鑰。

Q3 . 8 人純對稱兩兩通訊需幾把金鑰？
→ 8×7/2 = 28 把。
