考點：OSI 分層判斷

Exam: OSI Layer Identification
01 — 考前重點濃縮

D4 後期。本封考OSI 分層判斷——把協定/設備/攻擊歸到正確層,是 D4 最高頻考點。

L7 HTTP/DNS/SMTP;L4 TCP/UDP/埠;L3 IP/ICMP/路由器;L2 Ethernet/ARP/MAC/交換器;L1 纜線/集線器。
攻擊歸層:ARP 欺騙=L2、IP 欺騙=L3、SYN flood=L4、SQLi/XSS=L7。
記憶法(上到下):All People Seem To Need Data Processing。
02 — 歸層速記
項目層
路由器/IP/ICMPL3 網路
交換器/ARP/MACL2 資料連結
TCP/UDP/埠/SYN floodL4 傳輸
HTTP/DNS/SMTP/WAFL7 應用
03 — 關鍵名詞（中英）
OSI Layer（OSI 分層）— 七層參考模型。
ARP（位址解析）— IP↔MAC,L2。
Switch / Router（交換器/路由器）— L2 / L3 設備。
WAF（Web 應用防火牆）— L7。
SYN Flood（SYN 洪水）— L4(TCP)攻擊。
04 — 常見陷阱

ARP(L2) vs IP(L3) 欺騙別混;交換器 L2、路由器 L3。

WAF 是 L7(看應用內容);傳統防火牆多 L3/L4。

TCP/UDP 與埠在 L4。

05 — 🎯 實戰考題
QUESTION 1

At which OSI layer does ARP spoofing operate?

（ARP 欺騙在哪個 OSI 層運作?）

A. Layer 2 (Data Link) 資料連結層
B. Layer 3 (Network) 網路層
C. Layer 4 (Transport) 傳輸層
D. Layer 7 (Application) 應用層
✅ 解答與解析

正解：A。ARP 解析 IP↔MAC,屬L2(資料連結層);ARP 欺騙在 L2 進行 MITM。

為何其他選項錯：
  • B：IP 欺騙才在 L3。
  • C：L4 是 TCP/UDP/埠相關攻擊(如 SYN flood)。
  • D：L7 是應用層(HTTP/DNS)。
QUESTION 2

A SYN flood attack that exhausts a server's connection table operates primarily at which layer?

（耗盡伺服器連線表的 SYN flood 攻擊,主要在哪一層運作?）

A. Layer 2
B. Layer 3
C. Layer 4 (Transport)
D. Layer 7
✅ 解答與解析

正解：C。SYN flood 濫用 TCP 三次握手,屬L4(傳輸層)攻擊。

為何其他選項錯：
  • A：L2 是 ARP/MAC 相關。
  • B：L3 是 IP/ICMP。
  • D：L7 是應用層攻擊(如 HTTP flood,雖也算 DDoS 但機制不同)。
QUESTION 3

A Web Application Firewall (WAF) that blocks SQL injection operates at which OSI layer?

（阻擋 SQL 注入的 Web 應用防火牆(WAF)在哪個 OSI 層運作?）

A. Layer 3
B. Layer 4
C. Layer 7 (Application)
D. Layer 2
✅ 解答與解析

正解：C。WAF 檢視應用層內容(HTTP 請求/payload)以擋 SQLi/XSS,屬 L7。

為何其他選項錯：
  • A／B：傳統封包/狀態防火牆在 L3/L4,看不到應用內容。
  • D：L2 與應用內容無關。
06 — (ISC)² 選最佳答案思路

歸層:設備(交換器 L2/路由器 L3)、協定(TCP L4/HTTP L7)、攻擊(ARP L2/IP L3/SYN L4/SQLi L7)。

WAF=L7;傳統防火牆 L3/L4。先定『它處理什麼』再歸層。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：OSI 歸層:ARP L2、IP L3、TCP/埠 L4、HTTP/DNS/WAF L7;攻擊也依此歸層。

Q1 . ARP 欺騙與 IP 欺騙分別在哪層？
→ ARP 在 L2、IP 在 L3。

Q2 . SYN flood 在哪一層？
→ L4(傳輸層,濫用 TCP 握手)。

Q3 . WAF 在哪一層運作？
→ L7(應用層)。
