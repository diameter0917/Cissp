防火牆/IDS/分段深入

Firewalls/IDS/Segmentation Deep Dive
01 — 中期 D4：邊界與內部防禦

今天深入防火牆世代、IDS/IPS的偵測方式與誤報漏報、以及網路分段與零信任。重點：邊界防禦會被突破，內部分段才能限制橫向移動。

🗺️ 今天重點：防火牆世代/WAF → IDS vs IPS、特徵 vs 異常 → 誤報漏報 → 分段/微分段/零信任。
02 — 防火牆世代
世代檢視層級
Packet FilterL3/L4 標頭（IP/埠），無狀態
Stateful追蹤連線狀態（state table）
Proxy/ALGL7 代理，內容檢視
NGFW整合 IPS/應用辨識/使用者身分
WAF專護 Web 應用（L7，擋 SQLi/XSS）
03 — IDS / IPS
IDS 偵測並告警（旁路、被動）；IPS 偵測並阻擋（在線、主動）
Signature-based 特徵比對：準、但抓不到未知（zero-day）
Anomaly/Behavior-based 異常偵測：能抓未知、但誤報多
False Positive（誤報，正常被當攻擊）vs False Negative（漏報，攻擊沒抓到，更危險）

💡 IPS 在線可擋但誤報會擋到正常流量（影響可用性）；IDS 旁路只告警不影響流量。漏報(false negative)比誤報更危險——真攻擊溜過去。

04 — 分段與零信任

網路分段（VLAN/子網/DMZ）限制爆炸半徑；微分段（micro-segmentation）細到工作負載層級。零信任（Zero Trust）原則：永不信任、持續驗證，不因『在內網』就放行——每次存取都驗身分、裝置與情境（呼應最小權限）。

05 — 關鍵名詞速查
Stateful Firewall（狀態防火牆）— 追蹤連線狀態做判斷。
NGFW（次世代防火牆）— 整合 IPS/應用/身分辨識。
WAF（Web 應用防火牆）— L7 防 SQLi/XSS 等。
IDS / IPS（入侵偵測/防禦）— 告警 vs 阻擋。
Signature/Anomaly（特徵/異常偵測）— 已知準確 vs 抓未知但誤報多。
False Positive/Negative（誤報/漏報）— 正常誤判 vs 攻擊漏抓(更危險)。
Micro-segmentation（微分段）— 工作負載層級的細緻隔離。
Zero Trust（零信任）— 永不信任、持續驗證。
06 — 易混淆釐清

IDS vs IPS：IDS 旁路被動只告警；IPS 在線主動可阻擋（但誤報會擋到正常）。

False Positive vs Negative：誤報＝正常被當攻擊（擾人、可能擋到業務）；漏報＝攻擊沒被抓到（最危險）。調校在兩者間取捨。

Signature vs Anomaly：特徵抓已知（漏報未知）；異常抓未知（誤報多、需基線）。

Firewall vs WAF：傳統防火牆看 L3/L4；WAF 看 L7 應用內容（擋注入/XSS）。

07 — 條文與標準

NIST SP 800-41（防火牆）、SP 800-94（IDS/IPS）、SP 800-207（Zero Trust Architecture）。DMZ、分段、零信任皆為縱深防禦的網路落地。

08 — 真實事件：防火牆規則錯誤
🔎 一條規則開錯，門戶大開

許多外洩源於防火牆/雲端安全群組規則設定錯誤——把『僅限內部』誤設成 0.0.0.0/0（對全世界開放），或規則順序錯誤使預期的拒絕被前面的允許覆蓋。複雜規則庫長期累積、缺審查，是常見破口（呼應 D2 雲端 bucket 公開）。

教訓：防火牆需最小化規則、定期審查、預設拒絕（default deny）、變更管理；雲端尤其要自動化檢測公開暴露。規則即程式碼，要審查與測試。

📌 延伸：DMZ 與堡壘主機

DMZ（非軍事區）是放置對外服務（Web/郵件）的隔離網段，介於內網與外網之間，使外部只能觸及 DMZ、碰不到內網核心。堡壘主機（bastion/jump host）則是進入內部的單一受控跳板，集中強化與稽核。兩者都是『把信任邊界畫清楚』的經典分段手法。

09 — CISSP 考點提醒

選最佳答案：『偵測並阻擋』→ IPS；『只告警不影響流量』→ IDS；『擋 Web SQLi/XSS』→ WAF；『限制橫向移動』→ 分段/微分段；『不因在內網就信任』→ 零信任。

常見陷阱：漏報比誤報危險；防火牆預設應 default deny；零信任不等於某產品，而是原則。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：防火牆由封包濾→狀態→NGFW/WAF；IDS 告警/IPS 阻擋；特徵抓已知/異常抓未知；分段與零信任限制橫向移動、持續驗證。

Q1 . IDS 與 IPS 的關鍵差別？
→ IDS 旁路只告警；IPS 在線可阻擋（誤報會擋到正常流量）。

Q2 . 誤報與漏報哪個更危險、為什麼？
→ 漏報（false negative）——真攻擊沒被抓到。

Q3 . 零信任的核心原則？
→ 永不信任、持續驗證；不因位於內網就放行。
