安全治理與角色權責

Security Governance & Roles
01 — 治理是什麼、為什麼先講它

昨天我們立起 CIA 這根「目標」的柱子；今天要回答「由誰負責、怎麼確保它真的被做到」。這就是安全治理（Security Governance）。治理是把資安從「IT 部門的技術活」提升成「由上而下、與商業目標對齊的管理活動」——董事會與高階主管設定方向、分配資源、課responsibility，IT 只是執行末端。CISSP 反覆強調一句話：安全是管理問題，不是技術問題。

🗺️ 今天的地圖：治理三特徵（對齊/由上而下/問責）→ 四種角色權責 → due care vs due diligence → 治理 vs 管理。先建立「誰該做什麼」的骨架。
02 — 角色與權責（由上而下）

資安角色像一條責任鏈，由上而下授權、由下而上負責：

董事會 / 高階管理層 — 最終當責 (Accountable)
CISO / 安全長 — 制定策略、統籌計畫
資料擁有者 Data Owner — 分類、定義保護需求
保管者 Custodian / 使用者 User — 執行、遵循

💡 關鍵觀念：責任（Responsibility）可以下授，當責（Accountability）不能下授。高階主管可以把「做」交給別人，但「最後出事要扛」永遠在管理層。

03 — 四個核心角色一句話
角色 負責什麼 記憶點
Data Owner
資料擁有者		 決定分類等級、定義誰可存取 通常是業務主管，非 IT
Data Custodian
保管者		 依擁有者要求執行備份、加密、存取設定 通常是 IT / DBA
System Owner
系統擁有者		 對承載資料的系統安全負責 管「系統」而非「資料」
User
使用者		 依政策使用，遵守可接受使用規範 (AUP) 最小權限對象
04 — 關鍵名詞速查
Governance（治理）— 確保安全活動支持並對齊商業目標的監督機制。
Accountability（當責）— 最終承擔結果者，不可下授。
Responsibility（責任）— 實際執行任務者，可分派。
Due Care（應盡注意）— 採取一個審慎之人會採取的合理保護措施（「做了該做的」）。
Due Diligence（盡職調查）— 持續了解風險、調查與驗證的過程（「持續去了解」）。
AUP（可接受使用政策）— Acceptable Use Policy，規範使用者如何使用資產。
05 — 易混淆釐清：Due Care vs Due Diligence

記憶法：Due Diligence = 調查/思考（蒐集資訊、評估風險）；Due Care = 行動（依評估結果實際做防護）。先 diligence（了解）才能 care（動手）。

考試常以「公司是否善盡責任、能否免除過失責任」的情境出現。兩者都做到，才構成法律上的「prudent person（審慎之人）」標準，是日後爭訟時的關鍵抗辯。

06 — 條文與標準

治理有成熟的框架可循：ISO/IEC 27001 要求管理層展現「領導與承諾（leadership & commitment）」；COBIT 專注於 IT 治理與企業目標對齊；NIST Cybersecurity Framework (CSF) 以 Govern/Identify/Protect/Detect/Respond/Recover 把治理列為核心。台灣的金管會（FSC）公司治理與資安規範，也要求董事會層級對資安負責——精神一致：tone at the top（高層定調）。

07 — 真實事件：權責不清的歸屬爭議
🔎 Equifax 2017：當責回到了 CEO

2017 年 Equifax 因未修補一個已知漏洞（Apache Struts）導致約 1.47 億人個資外洩。事後調查暴露的核心問題正是治理失靈：修補的「責任」分散到工程團隊、但「當責」不清，內部沒有人對整體把關。最終 CEO、CIO、CSO 多人去職——印證了「當責不可下授」：縱使執行者是基層，最後扛責任的是管理層。良好治理會明確指定「誰對什麼當責」，避免人人有責變成人人無責。

08 — CISSP 考點提醒

高頻考點：題目給情境問「誰該負責決定資料分類？」答案幾乎都是 Data Owner（業務主管），不是 IT/Custodian。看到「決定/核准/分類」想 Owner；看到「執行/備份/設定」想 Custodian。

常見陷阱：Senior management 永遠是 ultimately accountable；別被「IT 部門應負責」的選項誤導。安全方案的成敗，CISSP 觀點一律歸到管理層的支持與資源。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：治理是「由上而下、與商業對齊、管理層當責」的監督機制；責任可下授、當責不可，Due Diligence（了解）先於 Due Care（行動）。

Q1. 誰最終對組織的資安成敗當責？
→ Senior management / 董事會（Accountability 不可下授）。

Q2. 決定一份財務報表的機密分類，是誰的職責？
→ Data Owner（該業務的擁有者），非 IT Custodian。

Q3. 公司事前做了風險評估、也據此部署了控制——這分別對應什麼？
→ 評估＝Due Diligence；部署控制＝Due Care。
