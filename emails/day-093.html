考點：雜湊與 PKI

Exam: Hashing & PKI
01 — 考前重點濃縮

本封考雜湊(完整性) vs 加密(機密性)、數位簽章、與 PKI 憑證鏈/撤銷(CRL/OCSP)。

雜湊=單向、驗完整性(密碼存加鹽雜湊,不是加密)。
數位簽章=私鑰簽、公鑰驗→完整性+不可否認;HMAC 用共享密鑰、無不可否認。
憑證撤銷:CRL(清單) vs OCSP(即時查單張)。
02 — 核心速記
機制提供
雜湊 Hash完整性(單向)
加密 Encryption機密性(可逆)
數位簽章完整性+不可否認+真實性
HMAC完整性+來源鑑別(無不可否認)
CRL/OCSP查憑證撤銷狀態
03 — 關鍵名詞（中英）
Hashing（雜湊）— 單向、驗完整性。
Digital Signature（數位簽章）— 私鑰簽公鑰驗;不可否認。
HMAC（雜湊訊息鑑別碼）— 共享密鑰;無不可否認。
X.509 / CA（憑證/憑證機構）— 綁定身分與公鑰。
CRL / OCSP（撤銷清單/線上狀態）— 查憑證是否被撤銷。
04 — 常見陷阱

雜湊≠加密:雜湊單向(完整性),加密可逆(機密性);密碼用加鹽雜湊儲存。

簽章私鑰簽/公鑰驗(與加密相反)。

HMAC 無不可否認(雙方共享密鑰皆能產生)。

05 — 🎯 實戰考題
QUESTION 1

To verify that a downloaded file has not been altered, a user compares its SHA-256 value. Which security property does this provide?

（為驗證下載檔案未被竄改,使用者比對其 SHA-256 值。這提供哪個安全性質?）

A. Confidentiality 機密性
B. Integrity 完整性
C. Availability 可用性
D. Non-repudiation 不可否認
✅ 解答與解析

正解：B。雜湊(SHA-256)比對驗證資料未被更動＝完整性(Integrity);雜湊不提供機密性。

為何其他選項錯：
  • A：雜湊不加密、不提供機密性。
  • C：與可用性無關。
  • D：單純雜湊無身分綁定,不提供不可否認(需數位簽章)。
QUESTION 2

Which provides BOTH integrity and non-repudiation?

（下列何者同時提供完整性『與』不可否認?）

A. Symmetric encryption 對稱加密
B. Hashing alone 單純雜湊
C. HMAC
D. Digital signature 數位簽章
✅ 解答與解析

正解：D。數位簽章用私鑰簽(只有持有者能簽)→提供完整性＋不可否認＋真實性。

為何其他選項錯：
  • A：對稱加密提供機密性,非不可否認。
  • B：單純雜湊只驗完整性,無身分。
  • C：HMAC 用共享密鑰,雙方皆能產生→無不可否認。
QUESTION 3

A server wants browsers to check a certificate's revocation status in real time for a single certificate. Which mechanism is MOST appropriate?

（伺服器希望瀏覽器即時查詢單張憑證的撤銷狀態。最適合的機制為何?）

A. CRL 憑證撤銷清單
B. OCSP 線上憑證狀態協定
C. HSTS
D. DNSSEC
✅ 解答與解析

正解：B。OCSP 即時查詢單張憑證狀態;CRL 是定期發布的整份清單(較舊、延遲)。OCSP Stapling 可再省查詢、護隱私。

為何其他選項錯：
  • A：CRL 是清單式、有延遲,非即時單張查詢的最佳解。
  • C：HSTS 強制 HTTPS,與撤銷查詢無關。
  • D：DNSSEC 保護 DNS,非憑證撤銷。
06 — (ISC)² 選最佳答案思路

雜湊=完整性、加密=機密性、簽章=完整性+不可否認。HMAC 無不可否認。

撤銷:即時單張→OCSP;清單→CRL。密碼儲存→加鹽雜湊(非加密)。

✏️ 一句話總結 + 自我檢測（點開看解答）

一句話：雜湊驗完整性、加密保機密、簽章(私鑰簽)加不可否認、HMAC 無不可否認;撤銷 OCSP(即時)/CRL(清單)。

Q1 . 比對檔案雜湊值提供什麼性質？
→ 完整性(Integrity)。

Q2 . 同時提供完整性與不可否認的是？
→ 數位簽章(私鑰簽、公鑰驗)。

Q3 . 即時查單張憑證撤銷狀態用什麼？
→ OCSP。
